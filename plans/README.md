## Dependency notes

- The plans are independent. Plans 001–003 should land before performance work
  because they repair user-observable Linux semantics.
- Plan 005 changes scoped interfaces and documentation that are already touched
  by unrelated working-tree changes. Its executor must reconcile, never replace,
  those changes.

## Findings considered and rejected or deferred

- Alternate signal-stack containment was rejected: for a downward-growing RISC-V
  stack Linux v7.1 intentionally defines the active interval as `(base, top]`;
  `AlternateSignalStack::contains_sp` already matches that fixed source.
- Full MAP_FIXED rollback was rejected: Linux validates file/backing permissions
  before replacement, but does not promise restoration after every later
  `mmap_region` failure. Plan 003 fixes the demonstrated ordering bug only.
- Per-page journal commits and scheduler stale-entry sweeps remain confirmed
  performance opportunities. Pipe byte-at-a-time copies, ext2 full-cache
  rollback snapshots, and descriptor-table quadratic scans are addressed by
  Plans 006, 008, and 009 respectively.
- Replacing three `lazy_static!` globals with `spin 0.10::Lazy` and deleting the
  stale nested `kernel/Cargo.lock` are worthwhile cleanup, but lower priority than
  the selected correctness and memory-cost work.
- README display/toolchain drift and broader architecture-check fixtures are real
  DX/test debt, but do not precede the selected kernel semantic fixes.
