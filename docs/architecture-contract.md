# LiteOS Architecture Contract

本文只定义稳定的 module interface、依赖和状态 owner。当前功能事实属于 `architecture.md`，Linux ABI 状态属于 `syscall-support.md`。

## 1. Crate contract

- `bootloader` 是独立 M-mode domain，不依赖 kernel 或 userspace。
- `syscall-abi` 只保存 kernel dispatcher 接入的 Linux/riscv64 ABI 常量，不依赖实现 crate。
- 产品 userspace 只有固定上游 musl + BusyBox rootfs，以及复用同一 musl runtime 的单 ELF `liteos-stress` 诊断命令；禁止恢复 Rust user crate、自有 runtime/init 或第二条默认镜像路径。
- kernel 的 `main.rs` 是唯一 composition root；初始化顺序和 adapter 装配不得下沉到 driver、filesystem 或 task。

## 2. Kernel dependency contract

| Module | 允许依赖（机器读取） | 说明 |
|---|---|---|
| `arch` | `config`, `memory`, `sync` | architecture mechanism 不得消费上层领域状态 |
| `config` | 无 | 只保存无运行时依赖的常量 |
| `fallible_tree` | 无 | 无状态的确定性 AVL mechanism；只提供显式 OOM 的有序节点 publication，不拥有领域数据 |
| `sync` | `arch` | 只依赖本地中断机制 |
| `memory` | `arch`, `config`, `fallible_tree`, `id`, `random`, `sync` | 不感知 task、filesystem 或具体 driver policy |
| `drivers` | `arch`, `fallible_tree`, `memory`, `sync` | 不感知 task、filesystem 或 syscall；interrupt registry 只使用通用 fallible ordered publication |
| `drm` | `drivers`, `fallible_tree`, `ipc`, `memory`, `sync` | 只消费通用 display seam；GEM handle 使用统一 fallible ordered publication；不感知 VirtIO adapter、task、filesystem 或 syscall ABI |
| `input` | `drivers`, `ipc`, `sync`, `timer` | 只消费通用 input seam，并拥有 evdev 事件域；不感知 VirtIO adapter、task、filesystem 或 syscall ABI |
| `ipc` | `id`, `sync` | 只拥有 Pipe byte/endpoint，不感知 fd、task、socket 或 syscall；`id` 仅分配 anonymous inode identity |
| `socket` | `drivers`, `fallible_tree`, `id`, `ipc`, `sync`, `timer` | 拥有 socket domain facade、AF_UNIX 与 AF_INET stack；`drivers` 只允许 network-device seam，`id` 仅分配 anonymous inode identity |
| `fs` | `drivers`, `drm`, `fallible_tree`, `input`, `ipc`, `memory`, `socket`, `sync`, `timer` | `drivers` 仅允许 `block` seam；`drm`/`input` 仅允许 OFD backend；socket 仅允许统一 OFD backend facade；`memory` 仅允许 shared-page seam |
| `task` | `arch`, `drm`, `fallible_tree`, `fs`, `input`, `ipc`, `memory`, `socket`, `sync`, `timer` | 不依赖具体 device 或 syscall/trap entry；只在 deferred context 推进 network、display 与 input work |
| `trap` | `arch`, `drivers`, `memory`, `syscall`, `task`, `timer` | 只做入口、分类和事件投递 |
| `syscall` | `drm`, `fs`, `input`, `ipc`, `memory`, `random`, `socket`, `system`, `task`, `timer` | DRM/evdev 只编解码标准 UAPI；不得绕过 facade 接触 adapter/scheduler/page table |
| `random` | `drivers` | entropy facade；只消费 RNG device seam，不生成伪随机 fallback |
| `system` | `arch` | 只拥有 whole-system reset/shutdown/CAD policy |
| `timer` | `arch`, `config`, `drivers`, `sync` | RTC adapter 由 timer 唯一拥有 |
| `log` | `arch`, `sync` | 日志策略和输出在本 module 内闭合 |
| `id` | 无 | 纯 ID allocation mechanism |
| `lang_item` | `arch` | 只使用 architecture fail-stop mechanism |
| `main` | `arch`, `config`, `drivers`, `drm`, `fallible_tree`, `fs`, `id`, `input`, `ipc`, `lang_item`, `log`, `memory`, `random`, `socket`, `sync`, `syscall`, `system`, `task`, `timer`, `trap` | 唯一 composition root |

同一 module 内引用不构成跨 seam 依赖。`main.rs` 可以依赖所有 kernel module，但只能做装配、启动顺序和 fail-stop 策略。

## 3. State owners

| 状态 | 唯一 owner |
|---|---|
| M-mode initialization、HSM、RFENCE | bootloader 对应 module |
| DTB board facts | immutable BoardInfo publication |
| raw hart possible/online/active、startup stack、同步 memory-barrier request/completion generation | HartTopology；有序 hart table 与同批发布的 immutable raw-ID → compact-index projection 唯一定义紧凑 index，每次调用的 generation 由 arch hart barrier mechanism 唯一分配 |
| per-hart current、runqueue、mailbox 与 Ready load projection | task ProcessorTopology；每个 live Thread 必须拥有固定 kernel stack，因此 topology 以物理页容量/kernel-stack 页数作为每 hart queue 与 mailbox 的可证上界并在启动时预留；`ready_entries` 只投影 SchedulingState 的 logical Ready membership，不拥有 heap/mailbox token；runtime delivery 不分配，stale generation 只在 heap root、mailbox batch 或 capacity pressure 下清理 |
| per-hart startup stack、softirq pending、timer deadline、memory-barrier generation 与 online/active publication | arch HartTopology；timer 只经 `advance_timer_deadline` 推进所属 hart 的固定相位 deadline；timer、console、network、display、input backlog 只使用各自合并 softirq bit，禁止从 handler 复制 deferred membership |
| task run state、generation、wait membership、wake result 与 per-Thread logical CPU affinity | SchedulingState；affinity 与 membership 同锁，Ready ingress/egress 必须产生线性 `ReadyTransition`/`ReadyRetirement` token 并由 processor 在同一 lock lifetime 内提交 per-hart projection，禁止按 syscall/CPU 复制 mask |
| per-Thread nice/vruntime、active CPU slice/dispatch weight、累计 runtime、legacy scheduler policy/reset-on-fork 与 I/O priority | Sched policy lock；fork/clone 只经 `Sched::forked` 派生，exec 保留；ioprio syscall 不保存 ABI shadow state |
| process address-space handle、cwd opened-entry、fd table、credentials/umask、resource limits、聚合 CPU runtime | Process；Thread 共享，fork 复制 limits 并重置 runtime，exec 保留；supplementary groups 以 immutable `Option<Arc<Vec<u32>>>` 发布，pathname/fork 快照只复制 Arc；vfork child Process 初始持同一 AddressSpace Arc，exec 只替换 child handle；最后一个 Thread exit 立即取走 fd table |
| fd slot vector、FD_CLOEXEC、descriptor publication 与空洞选择 | fs::file::descriptor_table 的 FileDescriptorTable；其 private IndexedSlots 是 slot vector 与四层 free-slot index 的唯一复合 owner，occupied-prefix lower bound 只是同 owner 内 slot emptiness 的派生搜索结构，不穿过 module seam；close/dup3 replacement/CLOEXEC 在锁内只 detach，entry Drop 与 epoll/flock/record-lock consequence 全部在锁外执行；exec 另用 transient cursor 和 32-entry fixed stack batch 单调扫描，每个 slot 只访问一次 |
| VMA 与 private expedited membarrier registration | AddressSpace；CLONE_VM/vfork 共享，fork/exec 新 owner 从未注册开始 |
| PID/TID allocation、parent edge、live thread collection、process-creation reservation、exec generation、group-exit status、job-control/orphan lifecycle 与 child event/waiter | TaskManager process graph；`pid::PID_MAX` 只允许 TaskManager allocator 与 robust-list owner codec 调用，单调 ID 不得超过 Linux `FUTEX_TID_MASK=0x3fffffff`，max+1 只作为 exhausted sentinel，后续 fork/vfork/thread clone 在发布前返回 `EAGAIN`，保证所有已发布 TID 可编码为 robust-futex owner；独立 creation lock 只覆盖 `RLIMIT_NPROC`/PID capacity 检查到 graph publication，防止并发超限；vfork child node 唯一持有 suspended parent Thread |
| ITIMER_REAL record 与 active deadline index | TaskManager 唯一 RealTimerQueue；fork 不继承、exec 保留、最后一个 Thread exit 删除。set/get/exit 固定 graph → timer 锁序；timer deferred 只持 timer lock 摘取与重装 deadline，释放后才经统一 signal seam 发布 SIGALRM |
| deadline/futex/pipe/poll/signal/console wait registration、event filter、exclusive mode 及其 indexes | TaskManager 唯一 IndexedWaitQueue；publication 在 SchedulingState 变为 Blocking 前预分配 entry 与全部 index node，失败原样返回 OOM；futex index 只消费 memory 归一化的 private/shared key，requeue 原地迁移同一 registration；blocking pipe writer registration 保存本次原子写所需容量，poll 仍只观察通用 endpoint readiness；ppoll/epoll/socket blocking 共用 source indexes；socket 内部 edge notification 固定注册为 source-native read key，userspace event mask 只用于真实 data source 和后续 level recheck；source wake 唤醒全部普通 callback group（每个 epoll instance 一个 thread）及一个 exclusive group |
| signal disposition、process-directed shared pending set | Arc<Process> 的单一 ProcessSignalState lock |
| signal mask、thread-directed pending set、alternate stack registration、active frame | ThreadContext 与用户 RV64 rt_sigframe；altstack active 只由用户 SP/range 推导，不复制 flag |
| interrupted syscall 的单次 replay record | ThreadContext；signal frame 保存最终 replay/EINTR 上下文 |
| OFD backend、opened-entry identity、offset/status flags、跨 fd-table descriptor 引用数 | OpenFileDescription；pathname-backed character/regular/directory fd 保留 VFS opened entry，anonymous pipe 使用 pipefs 语义；最后 descriptor close 触发 epoll interest cleanup |
| Linux/RV64 raw iovec layout、page-batched array import 与单次 scatter/gather progress | `syscall::user_iovec`；raw importer 只验证 count/null/array arithmetic 并按 userspace page 保序 copyin，不拥有 fs/socket total policy；`UserIoCursor` 唯一维护 index/offset/completed，stream stage 只读捕获 prefix，只有 backend 实际消费的 byte count 才推进 progress；caller-selected `project_total_length` 的 production caller 只允许 `syscall::socket::message`，`tools/kernel-unit` 只复用该纯函数验证 MAX_RW_COUNT prefix projection |
| socket atomic-message send/receive bounds 与 stream/atomic classification | `socket::message_limits`；`protocol` 只允许 parent `Socket` façade 把 domain/type 映射为 protocol policy，AF_UNIX/UDP/raw/packet backend 复用同一常量，syscall 不匹配 concrete adapter、不复制 limit |
| anonymous Pipe/AF_UNIX stream/PTY output 共用的 data byte ring、endpoint count、pipe `PIPE_BUF` atomicity、stream short-write policy 与内核 readiness notification token/generation | ipc::Pipe；data constructor 固定 64 KiB backing，notification constructor 固定一字节 backing，构造角色不保存为 runtime mode flag，也不复制到 fd table、socket state 或 wait registry；data transfer 只按最多两段连续 slice copy 并一次提交 head/length，禁止恢复逐字节取模或缓存 derived tail；notification token 只合并待消费 edge，每次 signal 仍推进 generation 并通知 registry，registry owner lock 内先排空 token、再做 socket level recheck |
| AF_UNIX abstract namespace、endpoint state、listen queue 与固定十消息 datagram queue | socket::UnixSocket；DatagramQueue 是容量与 full→non-full transition 的唯一 owner，不设 sender credit/byte shadow；OFD 只持统一 Socket Arc，fork/dup 共享，具体 domain adapter 不穿透 fs seam |
| Ethernet interface IPv4 address/prefix/default route、ARP cache、UDP/TCP/raw ICMP socket set、peer、IP_PKTINFO、socket option、ephemeral port、TCP listener backlog 与 FIN/TIME_WAIT orphan | socket::NetworkStack；ioctl、packet dispatch、procfs 只读同一 owner 快照；每个 endpoint 同时拥有 poll 前 readiness 与单一 pending-transition bit，锁外只发布 false→true edge；smoltcp socket storage 在 init 预留默认 `RLIMIT_NOFILE` 数量的 slot，满时在进入上游不可失败 `SocketSet::add` 前返回 `ENOMEM`；accepted handle 只转移给新 Socket/OFD，不复制协议状态；facade close 在原 endpoint record 标记 orphan，维护循环原地回收，不建立可扩容的析构 queue |
| AF_PACKET binding、protocol 与有界 receive queue | socket::PacketRegistry；RX frame 在 smoltcp ingress 前只镜像一次，packet endpoint 与 L3 NetworkStack 不复制彼此状态；device TX pool 0→nonzero 与 receive queue empty→nonempty 共用 endpoint pending edge，锁外通知统一 wait seam |
| VirtIO-net RX/TX queue、DMA slots、descriptor-head indexes、TX reservation/completion state 与 packet/byte counters | VirtIONetworkDevice 的唯一 queue lock；hardirq 只确认并发布 network softirq，deferred context 有界回收 completion、批量重新发布 RX 并唤醒 Pipe waiter；Reserved slot 只由 token Drop 取消，InFlight slot 只由 used-ring 归还 |
| VirtIO-GPU controlq、config event、display-info 候选 mode、交替 resource ID、command/response DMA storage、command fence、active/transition backing 与六阶段 scanout transaction | VirtIOGpuDevice 的唯一 control lock 与 backing Arc；启动期 bootstrap 完成后，hardirq 只确认并发布 display softirq，deferred context 每次只回收一个 completion并推进异步 display-info 或 CREATE→ATTACH→TRANSFER→SET→FLUSH→UNREF；每次 scanout 捕获所属 mode，旧 ID/backing 只在 UNREF completion 后归还，候选 mode 只在 DRM fallback allocation 成功后 commit |
| VirtIO-input immutable metadata、eventq、DMA slot、descriptor-head index 与 repost state | VirtIOInputDevice 的唯一 event queue lock 与 metadata；hardirq 只确认并发布 input softirq，deferred context 有界消费 raw event 并批量 notify repost，slot/head 映射损坏时 fail-stop |
| primary display adapter、DRM pending/completed fence、active framebuffer/mode、connector mode 与同代 fallback、master identity 与 completion notification | drivers::display 只发布唯一 `DisplayDevice` seam；drm::DrmDevice 唯一把该 seam、scanout backing lifecycle、mode 两阶段提交、master 与 Pipe edge 组合，task deferred 只调用 update façade，不复制 fence/master state；syscall 只消费 `DrmWait`/event 完成事实与 Pipe source后进入 task wait，不允许 drm 反向依赖 task |

| interrupt registration/affinity | interrupt controller |
| syscall number | syscall-abi |
| mounted inode 的 BSD advisory flock state | VFS；key 为 filesystem+inode，holder 为 OFD identity |
| mounted inode 的 POSIX byte-range record-lock state | VFS；key 为 filesystem+inode，holder 为 Process TGID；TaskManager 只拥有 interruptible wait membership |

新增 global、Atomic、lock、cache 或 flag 必须在声明附近写 `OWNER:`，并说明缺失该 owner 会造成的具体状态分裂。

## 5. Interface and capability contract

- kernel 与 bootloader 是 binary crate，跨 module interface 使用最窄的 `pub(super)`、`pub(in …)` 或 `pub(crate)`；不得使用裸 `pub` 伪造外部 interface。
- `memory::mm::fault_preflight` 拥有 `PageFaultAccess` 与 allocation-free fault classification，`memory::mm::mmap` 拥有 `PageFaultOutcome` 及其 retry postcondition；preflight 的 production caller 只有 `mmap::fault`，host test 只通过 path 复用同一实现。membership 与 U/R/W/X 拒绝必须在惰性 private-file EOF 投影前返回，EOF 必须在 `NeedsPrivateFrame` 前返回；否则非法 fault 会进入 frame allocation/global reclaim 并把 SIGSEGV/SIGBUS 错误升级为 OOM。`memory::mm::user_access` 唯一拥有完整 range fault-in、权限证明与 copy。`prepare_user_read` 只向 sibling `futex_key` 开放，使 key 归一化前取得同一 `U|R` 证明；write prepare 保持 private，因为 copyout/CAS 是唯一调用者。prepare 必须以已命中的 `user_page` 或 `PageFaultOutcome::Handled` 完成逐页权限证明，禁止恢复独立 `validate_user_range` 或 fault 后重复翻译；额外 validation pass 会让所有 user-copy 对每个页重复一次 Sv39 page-table walk，却不增加失败原子性。program break 由 `MemorySet` 单字段拥有，实际页复用 anonymous VMA；若恢复特殊 heap VMA，`MAP_FIXED`/`mprotect` 会绕过统一拆分语义。`mmap/mprotect` 必须接受 Linux `PROT_*` 合法组合（包括显式 W+X），ELF segment policy 留在 loader；在通用 VMA/PTE 层强制 W^X 会把 V8 JIT 的合法请求误报为 `EACCES`。`MAP_NORESERVE` 只在 syscall ABI seam 消费；无 swap/commit reservation owner 时不得向 VMA 增加 shadow flag，拒绝该标准 advisory flag 会让 V8 的 PROT_NONE reservation 在进入唯一 lazy mapping owner 前失败。
- `memory::mm::file_page_range::FilePageRange` 是 regular-file mmap offset/length 的唯一 checked projection；root construction 只允许 `FileMappingSource::new`，`SharedFileArea`、cached `PrivateFileArea`、split/sync/advice/fault/futex 只消费其 scoped methods，`tools/kernel-unit` 通过 path 复用同一实现。`SharedFileArea::sync_vma_range` 只供 `mmap::sync_shared_mapping`、`mmap::unmap_user_mapping` 与 `mmap::advice::sync_shared_before_discard` 使用，集中投影精确 byte range 与既有 writeback error；拆回三份会让 split 后 offset 语义漂移。`FilePageRange::stale_resident_start` 只供 cached `PrivateFileArea` 与 production-path host test 使用：它必须从原 mapping virtual origin 投影 EOF prefix，再 clamp 到 split VMA 起点，禁止把 prefix 直接加到当前 split 起点。`FileMappingSource` 从 syscall 直接穿过 TaskControlBlock/AddressSpace，后两层不得再接收 raw offset/length 或重建范围；invalid/alignment 映射 `EINVAL`，超过 RV64 `MAX_LFS_FILESIZE` 映射 `EOVERFLOW`，且两者都必须发生在 destructive `MAP_FIXED` unmap 前。page-cache 的 `SharedFileMapping::page` 实现是 file-fault snapshot seam：即使 cache hit 也先取得既有 per-inode operation lock，并在 cache node、shared frame 或 private frame 分配前按同一 truncate domain 判定 EOF；成功返回的瞬时 `Arc<dyn SharedPage>` pin 把 fault 线性化在后续 truncate 之前，不新增持久状态 owner。`PrivateFileArea::prepare_fault` 只由 sibling `mmap::fault` 在 `NeedsPrivateFrame` 之后、private frame allocation/reclaim 之前调用，`fill` 只消费该 snapshot，禁止在 allocation 后重新读取 live EOF；否则并发 truncate 可把 SIGBUS 路径变成全局 reclaim/OOM。`PrivateFileArea::first_stale_page` 只供 sibling `mmap::invalidate_shared_file` 投影同 backing 的 cached-private stale suffix；truncate callback 必须在同一次 AddressSpace lock 下同时删除 EOF 外 shared resident 与 cached private `data_frames` suffix，后者包含 snapshot 成功后、callback 取得 mm lock 前发布的 clean/dirty private 页，对齐 Linux `even_cows=1` 语义并使 fault-before-truncate 线性化最终可见。`memory::mm::mmap::fault` 只承接 `MemorySet` 的 fault transaction，保持原有 `pub(crate)` handle 接口供 trap/user-copy 调用；`resident::PrivateResident` 只向 mm sibling 暴露 frame/dirty/discardable 单记录，COW、mprotect、madvise、reclaim 与 statistics 是全部调用者。禁止重新建立并行 dirty/discardable tree，否则一次 residency publication 会恢复多个不可原子分配点。
- `fallible_tree::FallibleMap` 是需要 ordered publication 的 memory/drivers/drm/fs/socket/task owner 共用的唯一 fallible node mechanism：kernel AST 禁止 `BTreeMap/BTreeSet`，全部持久 `FallibleMap` 字段/static 必须与上表的 location+完整类型精确一致。新 key 必须先取得 `VacantEntry` 或 `NodeSlot`；`Box::try_new_uninit` 只能在 `try_reserve_node` 出现一次，checker 反向闭包 allocation call graph，除 `try_reserve_node/try_prepare/try_prepare_vacant/try_insert` 外的 rotation、remove、split、append、height-aware retain 与固定栈 iteration 均不得直接或间接分配。fallible-tree module 禁止 heap collection、其他 Box allocator、allocation method 与不透明 macro。retain 必须在任意批量删除后递归 join 高侧，单次 rotation 不能修复高度差大于 2 的树。`NodeSlot` 唯一拥有未初始化的 Box storage，`fill` 完整写入 `Node` 后按值消费 token 才允许 `assume_init`；若在 fill 前观察/析构为 Node 会触发未定义行为，若绕过 preflight 直接发布领域状态则 OOM 时会留下无索引 owner。树只拥有节点 topology，key/value 的领域状态仍由调用 module 唯一拥有。
- `IdAllocator` 的 recycle backing capacity 必须不小于从 initial 起历史签发的 fresh ID 总数；fresh publication 先按该总数 fallible reserve，recycled ID 再分配不改变上界。只按 `recycled.len() + 1` reserve 会让连续创建的 live kernel stack 不占容量信用，并在批量析构的不可失败 `dealloc` 路径触发 allocator abort。
- `ipc::EventFd::new`、`socket::UnixSocket::new` 与 `Socket` facade constructor 必须传播 control-block OOM，由对应 syscall 唯一映射 `ENOMEM`；Internet endpoint 在协议 poll 前冻结 readiness，poll 后只把 false→true 记录为 owner-local pending bit，再按稳定 ID cursor 锁外通知。`socket::packet::deliver` 只向 Ethernet device ingress 开放，queue 与 payload 在发布前完成 fallible reserve，只有 empty→readable 设置同类 pending bit；禁止构造大小随 socket 数增长的 notification Vec，也禁止把长期 writable/readable 当成每轮新 edge，否则会形成 network-softirq wake storm。`socket::inet::tcp::lifecycle::drop_endpoint` 只由 `InetSocket::drop` 调用，connected endpoint 原地转为 orphan，禁止恢复需要扩容的析构 queue。
- `fs::file::descriptor_table` 是 fd slot、FD_CLOEXEC、descriptor_refs publication、free-slot search 与 procfs snapshot 的深 module；嵌套 `fs::file::descriptor_table::indexed_slots::IndexedSlots` 是 slot vector 与 index 的唯一复合 owner，其 scoped interface 只允许直接 parent `fs::file::descriptor_table` 作为生产 caller，`tools/kernel-unit` 仅通过 path 复用同一实现做 host test，禁止其他 kernel module 直接调用。`IndexedSlots` 必须在发布 fd entry 前完成 slot vector 与四层 `FreeSlotIndex` 的全部 backing reserve，且两者使用摊销式 capacity growth，禁止以 per-slot `try_reserve_exact(1)` 把连续 open 重新退化为复制 O(FDSize²)；value constructor 只能在 limit 检查与全部 reserve 成功后执行。`FileDescriptorError` 唯一区分不存在的旧 fd、进程 fd 上限与 allocator OOM，syscall 分别映射 `EBADF`、`EMFILE` 与 `ENOMEM`。level 0 每个 set bit 精确表示一个 materialized empty slot，level 1–3 每个 set bit 精确表示 child word 非零；index 内唯一 `occupied_prefix` 只证明其下 materialized slots 全 occupied，in-order publication 只推进一个已证明 slot、detach 取 min 回退、fork 与 bitmap 同时复制，过低最多多做 bounded lookup，禁止越过未证明 hole。`MAX_FILE_DESCRIPTORS=1_048_576` 与 RV64 64-bit word 固定 16,384→256→4→1 words，所以 `allocate/allocate_pair/F_DUPFD(min)` 最坏只上行四层、下行三层，读取至多七个 bitmap word、执行至多四次 `trailing_zeros` 后返回 `>= min` 的最低空闲 fd，不得扫描 entry vector；dense in-order open 的 prefix 等于 slot length，lookup 不读 bitmap。index 只能作为同一 `IndexedSlots`/table/files lock 下的派生搜索结构：growth 必须先 reserve 所有 level 与 slots，OOM 不改变逻辑长度；empty↔occupied、fork clone、take-all 必须同时转移 index，局部 transition（含 dup3 occupied replacement）对旧 bit 做 fail-stop 断言。禁止在 Process、syscall 或 caller 保存第二份 hint/bitmap/free-list，也禁止把 index 暴露穿过 module seam。`DetachedFileDescriptor` 只允许 `task::model::file_descriptions` 的 close、dup3 replacement 与 exec CLOEXEC cleanup 消费：files lock 内只摘除 entry，`finish_close` 必须在 guard 释放后执行最后 descriptor 的 epoll/flock cleanup，再按既有顺序释放 Process record locks。exec 已拒绝多线程 Process，因此 transient cursor 在 cleanup 期间不会遇到本 Process 并发 fd publication；固定 32-entry stack batch 必须在复用前完全消费，使 CLOEXEC slot scan 为 O(FDSize)、files-lock acquisition 至多 `ceil(closed/32)+1` 且不分配，该 cursor/batch 不得持久化。OFD control block constructor 同样只允许 fallible publication；把 reserve 失败折叠为 `EMFILE` 会把资源上限与内核内存压力混成错误 ABI。
- `Credentials` 唯一拥有 supplementary groups 的 immutable `Option<Arc<Vec<u32>>>`；permission snapshot、fork 与 pathname walk 只能复制 Arc，空集合不得分配。`setgroups` 在 credential lock 内完成权限判定，锁外构造唯一新 Arc，成功后一次替换；构造失败必须返回 `ENOMEM` 且不改变旧 credentials。
- `task_manager::scheduler_nice` 的 `OutOfMemory` 只表示 target snapshot 在 graph lock 内无法取得 storage；scheduler syscall 唯一映射 `ENOMEM`，且失败前不得修改任一 target nice。其余 caller 不应观察 allocation adapter。
- `model::alternate_signal_stack` 是 syscall、signal delivery 与 sigreturn 共用的唯一 Thread altstack seam：syscall 只编解码 RV64 `stack_t`/errno；module 内部唯一执行 `SS_ONSTACK` 动态投影、RISC-V frame placement、nested overflow、`SS_AUTODISARM` commit/restore 与 fork/clone/exec lifecycle。禁止保存第二个 active flag，禁止 signal_state 自行计算 stack top；前者会与用户 SP 分裂，后者会绕过 overflow 和 autodisarm commit 时点。
- `processor::job_control::request_reschedule_on` 只向 parent scheduler 的 raw-target control path 开放；它映射一次目标并复用 processor 私有的 `publish_reschedule_at` 提交点。Ready delivery 已持有 verified compact index 时直接复用同一提交点；两条路径都必须统一发布 per-hart flag，并在远端用同一次 SBI IPI 唤醒 mailbox，其他 module 不得直接调用。`request_tick_reschedule` 只向 task deferred timer consumer 开放，并且只有本 hart 同时存在 Running 与 logical `ready_entries` 时才发布 reschedule；physical stale token 不得制造单任务自我切换。signal publication 若未消费 wait membership，必须显式请求 Running target reschedule，禁止再依赖 timer 的偶然抢占。
- `processor::ready_queue::{add_ready_entry,select_task,drain_inbound_to_local,deliver_ready_entry}` 的 `pub(super)` 只允许 parent `Processor` facade 的同名入口或私有 delivery 入口调用；它封闭 local heap、mailbox、generation validation、root pruning 与 capacity compaction，不得向 job-control/placement 泄漏 `Processor` 字段或容器 adapter。local add 以同一次 current-slot lookup 返回既有 Running ownership，delivery 禁止为 reschedule 判断重复解析 topology；local/remote slow body 分开 outline，capacity compaction 与连续 stale-root slow path 必须留在 cold section。嵌套 `task::scheduler::cfs_scheduler::preallocated_heap` 的 scoped interface 只允许直接 parent `cfs_scheduler` 作为生产 caller，`tools/scheduler-unit` 仅通过 path 复用同一实现做 host test；ordinary spare-capacity `make_room` 禁止调用 retain predicate。
- `processor::ready_membership::{commit_ready_transition,commit_ready_retirement}` 的 `pub(super)` 只允许 parent processor orchestration 与 sibling `placement/job_control/ready_queue` 调用；它们分别是消费 `model::scheduling::ReadyTransition` 与 `ReadyRetirement` 的唯一 seam。Rust 无 sibling-only visibility，故 token 的两个 `consume_ready_projection_*` 最小可见域仍为 `task`；architecture AST fence 必须按完整 AST scope 把 method、UFCS/path、alias 与 macro token 的唯一 caller 分别锁定到上述两个 top-level commit 函数，并要求每个 consumer/committer 恰好定义、调用一次。consumer 必须在 commit 的首个 reviewed statement 同步求值，committer 名称不得被 local/item/import 遮蔽；所有 `transition_to_ready`/`transition_ready_to_*` 又必须作为对应 commit 的直接参数，禁止通过 closure、临时变量、`mem::forget`、`ManuallyDrop` 或 macro 绕过 Drop fail-stop。其他 module 不得直接写 `ready_entries` 或拆开 run-state/count transaction。
- `ProcessorTopology` 构造时必须以物理页容量除以每个 Thread 固定 kernel-stack 页数，得到 live Thread 的保守上界，并为每个 local heap 与 inbound mailbox 一次性预留该容量。local enqueue 只在 capacity 不足时原地 retain，随后只弹出连续 stale root 以发布 live vruntime floor；remote delivery 只在 mailbox 已满时 retain。mailbox drain 对本轮 snapshot 只做一次全量 validation pass，并且仅在 surviving batch 将越过 local backing 时 compact heap；合并后只允许按 scheduling-visible root 复查连续 root，不得再扫描 batch。drain 必须保留 backing capacity；恢复 per-enqueue/per-delivery full sweep、空 `VecDeque` swap 或未预留 push 会分别把 wake burst 退化为 O(N²)、丢失 backing 或在 IRQ/wake 路径触发 alloc error。`SchedulingState::run_state` 必须保持 private；task 内只读 caller 使用 `run_state()`，非 Ready 两侧的既有状态机 caller 使用 `replace_non_ready_state()`，后者发现任一侧为 Ready 必须 fail-stop。`transition_to_ready` 与 `transition_ready_to_{running,stopped}` 分别返回不可复制且借用该 `SchedulingState` 的 `ReadyTransition<'_>`/`ReadyRetirement<'_>` token；processor 必须在同一 scheduling-lock lifetime 消费它们，按 New/Blocked/Stopped/Preempting→Ready 增计、Ready migration 先增 target 后减 source、Ready→Running/Stopped 减计。token 未消费、计数上下溢或 compaction 后 live token 仍超过容量均 fail-stop。`ready_entries` 是唯一 per-hart logical Ready projection；Relaxed reader 允许看到迁移两次原子操作间保守的瞬时多计，但不得漏掉稳定 membership，也不得用 physical heap/mailbox length 参与选核或 tick preemption。
- `model::scheduling::Sched::new` 只向 Process/Thread constructor 开放；`begin_runtime` 只由 `switch_to_task` 开始 active CPU slice，`checkpoint_runtime` 只由本 hart timer deferred consumer 提交已消耗增量而不结束 slice，`finish_runtime` 只由当前 task 的 block/preempt/vfork/exit orchestration 提交剩余增量并结束 slice。checkpoint 必须同步推进 Thread/Process/hart 累计与 Q10 vruntime，并保留 dispatch nice 快照；若只刷新 per-hart counter，会复制 runtime owner并在后续 finish 重复计费。`TaskControlBlock::cpu_runtime_snapshot` 只供 `clock_gettime` 与 same-thread-group procfs 只读投影 checkpoint 后的 active delta，不提交 runtime；`thread_statistics` 只向 task façade 的 procfs 采集开放，calling Thread 包含 active delta，其他 hart 上 running Thread 保持最多一个 scheduler tick 的 bounded-stale 累计值。Process 聚合 runtime 仍只读取其唯一 owner。nice scaling 固定使用 Linux v7.1 的 40 项 reciprocal 表并在热路径以乘法/移位推进，禁止复制权重表或退化为分档/运行时除法。重复 finish/checkpoint 必须 fail-stop，snapshot 不得写回或累加 live Thread 私有值，否则同一时间片会被重复观察，且 exited Thread 的 Process runtime 会丢失。
- `timer::monotonic_resolution_ns` 只向 `clock_getres` 开放，直接从 immutable DTB timebase frequency 换算且不缓存第二份 resolution；CPU clock resolution 由 scheduling 的微秒记账单位决定。`scheduler_quantum_ns` 只向 TaskManager scheduler interval seam 开放，从 timer owner 已校准的真实 tick interval 换算，syscall 不得复制 `TICKS_PER_SEC`。`arch::hart::advance_timer_deadline` 只向 timer interrupt programming 开放，并在内部选择 calling HartState；所属 hart 从上一 deadline 跳过已错过周期，避免 handler 延迟累积成相位漂移，timer caller 不得取得具体 state adapter。
- `timer::realtime_deadline_to_monotonic_ns` 只向 absolute realtime `clock_nanosleep` 开放，使用 timer owner 的 immutable boot offset 做单次精确域转换；syscall 不得先后采样两只时钟再估算，否则采样间隔会被错误加入 deadline。
- `task_manager::sleep_until` 只接受 absolute monotonic deadline 并返回领域 `WaitResult`；`nanosleep/clock_nanosleep` syscall 唯一负责 clock selector、relative/realtime 换算、remaining copyout 与 errno。task 层不得返回 ABI errno 或解释 userspace timespec，否则两个入口会形成独立 deadline/restart 语义。
- `task_manager::context_switch::prepare_current_block` 是 Running → Blocking 的唯一 publication seam：caller 把覆盖 readiness/signal 复查的 registry、process graph 或 unit owner guard 整体移交，closure 只发布对应 waiter 并返回 `WaitMembership`；seam 在返回 `PreparedBlock` 前提交 active runtime、取走 current、在 scheduling lock 下绑定 membership/state，并强制释放 owner guard。`PreparedBlock::suspend` 消费 token 一次完成 context switch 与 wake result 获取；未消费 token 的 Drop 必须 fail-stop，因为此时 current 已撤销且 task 会永久停在 Blocking。任何 caller 手写该序列都会重新引入 wait entry/state 分裂、runtime 重复提交或持 owner lock 切换导致的永久死锁。
- `task_manager::scheduler_affinity` 是 syscall 到 scheduler 的唯一 affinity seam：内部选择全局 live TID、执行 credentials permission、规范化 active mask、替换 Ready generation、请求 Running migration并同步等待 ownership 释放；syscall 只编解码变长 cpumask/errno。拆开会让 set/get、wake 与 migration 分别维护不同 mask，或在 syscall 成功后继续于禁止 CPU 执行。
- `task_manager::scheduler_policy` 是 legacy scheduler policy/param syscall 的唯一 seam：内部选择全局 live TID、保持 Linux target/validation/permission 顺序，并在 Sched policy lock 下查询或替换 reset-on-fork；相邻 `scheduler_rr_interval` seam 只完成同一 TID selection 后读取 timer owner 的固定 `SCHED_OTHER` quantum。syscall 只编解码 `sched_param`/`__kernel_timespec`、纯 priority bounds 与 errno。`thread_selector::scheduler_thread` 只供 policy/interval/affinity seam 复用零/current 与全局 TID 选择；`TaskControlBlock::scheduler_privilege_for` 也只向 policy/affinity 发布同一次 caller credential snapshot。`Sched::reset_on_fork` 只供 policy seam 查询/替换，`Sched::forked` 是 Process/Thread constructor 消费该 flag 的唯一入口；绕过会让 get policy 与 child nice 继承分裂。当前 setter 只接受 `SCHED_OTHER`/priority 0；没有独立 scheduling class 时不得开放 RT/BATCH/IDLE/DEADLINE/EXT setter，固定 interval 也不得宣称 Linux EEVDF dynamic entity slice 或 RT RR class。
- `task_manager::scheduler_nice` 是 get/setpriority 的唯一 collection seam：内部解释 PROCESS/PGRP/USER 零 selector、复制 live Thread owner、执行 effective-UID/root 与 target RLIMIT_NICE 判断，并保留 Linux 多目标部分成功及后续错误覆盖语义；syscall 只映射 selector、raw `20-nice` 与 errno。`Sched::nice` 只向该 seam 和 scheduler/procfs projection 开放；active slice 必须冻结 dispatch nice index，修改后经统一 reschedule seam 尽快切出，禁止按新权重追溯计费已运行时间。
- `task_manager::scheduler_io_priority` 是 `IOPRIO_WHO_PROCESS` query/set 的唯一 target、credential 与 policy mutation seam；syscall 只校验 encoded class/data 和映射 errno。`Sched::io_priority` 只向该 seam 开放，fork/clone 必须经 `Sched::forked` 继承，exec 保留；尚无 block scheduler enforcement 时不得声称 RT/BE/IDLE 改变实际 I/O dispatch。
- `arch::hart::hart_index` 是任意 raw hart ID 到紧凑 topology index 的唯一 O(1) projection；`current_hart_index` 用同一次 validated current-entry lookup 直接返回 calling index。其 fixed-width immutable rank table 只与有序 HartState table 一次性发布。ProcessorTopology 必须按同一顺序构造：current slot 与 local runqueue 操作直接使用 `current_hart_index`，选核扫描直接使用已有 candidate index，只有远端 raw target 才使用 `hart_index`，且一次映射必须同时定位 HartState 与 Processor slot；禁止再建映射、组合 `hart_id()+hart_index()`，或在 trap/scheduler 热路径扫描、二分 hart table。system façade 的 `getcpu/riscv_hwprobe`、SchedulingState 的 `CpuAffinity` 与 procfs CPU/processor 字段必须共用该顺序；SBI/PLIC/scheduler mailbox 继续使用 raw hart ID。`mark_online/mark_active/advance_timer_deadline` 由 arch façade 内部选择 calling HartState，具体 mutation method 保持 private。混用会在稀疏 DTB 上暴露不存在的 CPU number或把任务投递到错误 hart；分离可变 projection 会把 raw ID 路由到错误 per-hart owner。

- `task_manager::wait_registry` 的 scoped interface 只允许 parent orchestration、`prepare_current_block` 的 publication closure 与 sibling signal cancellation 使用；它唯一执行 membership/index 的 prepare/insert/remove/take，caller 不直接修改 `entries` 或 source index。entry 与全部 source/deadline index node 必须在进入 Running → Blocking publication 前准备完成，closure 内只执行不分配的 commit；否则 OOM 会留下已 Blocking 但不可唤醒的 Thread。timer deferred owner 在一次 registry lock 内最多摘取 32 个到期 entry，再锁外完成 scheduling wake；仍到期的 backlog 必须发布合并 `TIMER_BACKLOG_SOFTIRQ` 续批。逐 entry 重新抢锁会放大关中断开销，无界 drain 会饿死 I/O/user-return，等待下一 hardware tick 则会人为增加最多一个 quantum 的 timeout 延迟。

- `fs::Epoll::recheck_changed` 只允许 `syscall::poll::wait_keys::PollWaitGuards` 在 `wait_for_poll` registry-lock closure 内调用；它必须只原子 drain ctl/close token 并比较 caller 捕获的 Pipe read generation。generation 不同即中止本次入睡并让外层重建 keys，即使 token 已被另一份并发预建 snapshot 消费。该 interface 内禁止 snapshot、Arc clone、Vec、fallible operation 或 wait-key expansion；current level readiness 仍由 OFD/Epoll owner-local projection 独立复查。OOM 只能发生在进入 registry owner lock 前并返回 `ENOMEM`，不得折成 not-ready。`syscall::poll::wait_keys::{PollWaitKeys,PollWaitGuards}` 的 concrete module 保持 private，经 parent `poll` 只向 sibling `epoll` 重导出；`PollWaitKeys::{new,add_interest,add_socket_source,add_epoll_change_source,finish}` 与 `PollWaitGuards::changed` 是完整 interface。`add_socket_source` 的 scoped caller 只有同 builder 的 OFD expansion 与 parent poll 的 opaque peer-send wait；它不得接受 concrete socket adapter。一个 builder 唯一拥有一份 transient key Vec 与一份 shared snapshot-guard Vec，OFD 与最多五层 nested epoll 只能原地追加；每个 epoll 必须先捕获 notification generation、再 snapshot interests，connected datagram 则在同一次 source snapshot 捕获 opaque `SocketWaitGuard`。该 guard 的 constructor 只允许 AF_UNIX adapter 调用，`changed` 只允许 `PollWaitGuards` 在 registry owner lock 内调用；它只做 allocation-free Weak identity compare。`PollWaitGuards::changed` 必须遍历全部 guard、drain/compare 全部 epoll generation 并检查全部 peer identity，任一变化都中止入睡并重建 keys。`syscall::poll::prepare_wait_sources` 是 poll/epoll/direct blocking 共用的唯一递归 adapter-preparation seam；首次非阻塞 level evaluation 可在尚无 publication candidate 时直接调用，每次准备进入 `wait_for_poll` 的 publication invocation 则必须位于 keys/guards 构造后、registry owner lock 前。`PrepareIo::{Ignore,Propagate}` 唯一表达 poll/direct 保持既有 console-error policy 与 epoll `EIO` policy 的差异，nested snapshot OOM 一律传播。顶层与 nested epoll 的 ctl/close notification source 必须使用不分组的普通 registration，使一次变更先摘除全部已发布 stale membership；普通 data source 才继承顶层 instance wake-group。禁止恢复 per-interest/nested-edge 临时 key Vec、重复 concrete-adapter switch、在 publication recheck 重建 keys，或让 syscall 读取 epoll state/notification occupancy。

mutation scope 必须先更新本 interface contract 与固定容量证明，不得静默回退 heap map/Vec。`MutationGuard::begin_after` 只允许 ext2 metadata 在持有唯一 mutation lock、尚未发布 active transaction 时读取 live inode state；prepare None。regular read atime 只按 Linux relatime 条件进入该 domain，禁止每个 cache miss 都创建 metadata transaction。禁止 syscall/VFS 复制 link count、journal 状态或用写序调整冒充跨块原子性。
- `inode::StorageWriter`、`Inode::write_storage_batch` 与 `Inode::try_write_storage_batch` 是 page cache 向 filesystem adapter 提交固定 writeback batch 的唯一 seam：writer 只能在 callback lifetime 内顺序接收 offset/bytes，不暴露 ext2 guard、block map 或 journal；普通 batch 的默认 adapter 只为既有只读/volatile adapter 逐次复用 `write_storage`，try-batch 默认零副作用返回 `Busy`。ext2 覆盖必须让 callback 的全部写入共享一个 `MutationGuard`，只按最大 committed end 更新一次 inode size/mtime/ctime；try-batch 无法立即取得 mutation owner 时不得消费 callback，并统一返回 `Busy`。`ext2::storage_mutation::write_bytes` 只允许 parent 的 `write_storage` adapter 调用；`write_batch` 只允许 parent 的 `write_storage_batch` adapter 与同 module 的 `write_bytes` 调用；`try_write_batch` 只允许 parent 的 `try_write_storage_batch` adapter 调用；`write_at_locked` 只允许同 module append 与 sibling directory 的 symlink mutation 调用；`ensure_block_mapped` 只允许 sibling directory 与同 module 的 storage/allocate path 调用。提供完整 caller image 的对齐 storage write 若新分配 data block，必须直接以该 image 初始化；已映射 block 必须直接覆盖 journal image。partial data block、新 pointer block，以及 fallocate/目录语义要求的 data-block zero initialization 可以分配清零 scratch。`ext2::journal_layout::JournalLayout` 只允许 sibling journal 构造/读取，`tools/kernel-unit` 仅 path 复用同一 production formula；它按 descriptor 首 tag UUID 与保留 super/commit block 精确推导 unique home-block capacity。Journal stage 对既有 block 必须原地覆盖唯一 Vec；新 block 达 capacity 或 pressure-writeback staging OOM 必须在 durable commit 前返回并允许 batch 缩小重放。commit 发布 dirty journal state 后只能使用一个 PAGE_SIZE stack scratch，按阶段复用为 descriptor/escape，禁止再出现 heap OOM 半事务。layout 过高会越过 journal block map并 fail-stop，过低只允许提前拆 transaction，不得产生第二容量 owner。
- live mount root/mountpoint 不得被 unlink 或作为 rename source/replace target；VFS 在进入 adapter mutation 前返回 `EBUSY`，防止 mount table 与 opened-entry namespace 分裂。
- VFS permission evaluator 消费 Process 发布的 immutable identity snapshot，唯一决定 traversal、inode rwx、parent mutation、sticky directory、protected hardlink、setgid-directory inheritance 与 owner/mode change policy；syscall 和 filesystem adapter 不得复制权限规则。chmod/chown syscall 只把 ABI 值解码为 `OwnerModeChange`；ext2 在唯一 mutation lock 内把同一 inode 的 live type/mode/UID/GID 交给 evaluator，取得完整 update 后才发布 journal transaction，按需捕获该 inode 的单一 rollback preimage，并持久化 mode/UID/GID/ctime。授权失败不得进入 snapshot/journal，授权成功到 commit 之间 mutation lock 不得释放；否则前者形成拒绝路径放大，后者恢复 stale-owner TOCTOU。read-only inode 默认入口对 immutable metadata 调用同一 evaluator 后返回 `ReadOnly`，保留权限错误顺序而不复制 policy。
- `syscall::fs::io::sequential` 是 `read/readv/write/writev` 的唯一 external seam；四个 wrapper 只完成 fd/access、raw iovec import、fs-owned SSIZE_MAX policy 与 scalar-one-vector projection，所有 OFD kind dispatch 分别只存在于 internal `read::read_descriptor` / `write::write_descriptor` seam。恢复 scalar/vector 双轨会让 socket scalar I/O 再次受固定 staging 长度截断，并让 error/partial/signal 语义漂移。`syscall::user_iovec::import_iovecs` 是 fs 与 socket 共用的唯一 raw RV64 iovec import seam；连续 ABI array 必须按 PAGE_SIZE staging 批量 copyin，禁止恢复逐 iovec user-copy/page-walk 或在 raw importer 中加入 protocol/SSIZE_MAX policy。`UserIoCursor` 唯一维护 scatter/gather 的 vector index、vector 内 offset 和 completed byte count；stream gather 必须先只读 stage 完整 prefix，再只按 backend 实际返回值 commit，copy fault 的 staged prefix 不得提交，caller 禁止复制 progress 字段或让 short send 跳过未发送 suffix。socket syscall wrapper 按 Linux `MAX_RW_COUNT` 把 scalar/vector request 投影为一个有效 prefix：单 entry 在 range check 前截断，multi-iovec 先验证原始 ranges 再把越界 suffix 截断/置零；不得把该上限变成 EMSGSIZE 或改动 fs 已冻结的 SSIZE_MAX policy。`socket::message_limits` 是 atomic protocol 大小与 stream/atomic classification 的唯一 policy owner；其 `protocol` 只供 parent `Socket` façade 的 validate/send-capacity/receive-capacity projection 调用。`Socket::validate_send_length` 只供 sequential write、socket message send 以及 facade 内部 `write/send_to` 调用，`Socket::stream_send_staging_capacity` 只供 sequential write 与 socket message send，`Socket::receive_staging_capacity` 只供 sequential read 与 socket message receive，caller 只能消费这两个 capacity projection，禁止再次匹配 `SocketType`。sendmsg/sendto 在 payload gather 前经 Socket façade preflight，AF_UNIX/UDP/raw/packet 只分配已验证的完整 message；stream sendmsg/sendto/recvmsg/recvfrom 复用固定 64 KiB heap staging，不按 request total 分配。首次 user-copy/block/error 返回 errno，已有 backend progress 返回精确 partial；BrokenPipe 只在零 progress 且未设置 `MSG_NOSIGNAL` 时投递 SIGPIPE。eventfd 的 scalar/vector read 都遵循 Linux `eventfd_read` 的统一 iterator contract：总 capacity 至少八字节并只 scatter 一个 u64；eventfd writev 保持 scalar write fallback，逐个非空 iovec 要求恰好八字节，禁止把跨 iovec prefix 合并成 value。
- `syscall::fs::io::sendfile` 的 regular→regular scope 必须只消费 page-cache `RegularFile` facade：输入/输出各解析一次，唯一 PAGE_SIZE kernel staging 跨 chunks 复用，禁止递归调用 userspace read/write 或按 count 分配 buffer。两个不同 OFD 的 offset 必须按 OFD identity 全局排序后整次持有，输出 `RegularFileWrite` 必须覆盖完整 copy；缺失前者会让反向 sendfile ABBA，缺失后者会让其他 writer 穿插。显式 offset 只推进 userspace value，空 offset 只推进输入 OFD；输出 OFD 始终按实际 written bytes 推进。同一 backing 的重叠区间和 O_APPEND 必须拒绝；socket/pipe zero-copy seam 未建立前返回 `EINVAL`，禁止伪造支持。
- 只读动态 inode 必须经 `Inode::is_volatile` 在 `RegularFile` façade 内直接读取每次快照并拒绝 page-cache mapping；当前唯一 caller 是 procfs adapter，缺少该标记会冻结重复 proc 采样，持久 filesystem 不得设置。
- regular-file cache miss fill、`fallocate`、write/append/truncate 与 mmap writeback 必须共用 page-cache operation lock；ordinary read hit 只读 page index，miss 在取得 operation 后二次检查并唯一执行 storage read+publication，禁止旧 fill 穿过 mutation 后发布 stale page。mmap fault 即使 hit 也必须先取得 operation、在 node/frame allocation 前验证 page start 仍低于 EOF；返回的 Arc 是 fault-before-truncate 的线性化凭据，private fault 必须在其后才分配 frame，shared fault 必须在其后才采用 resident PTE。`SharedFrame::bytes_mut` 只允许 page cache 在 `Arc<CachedPage>` publication 前直接填充独占 miss frame；其他 caller 不得用它绕过 shared-page 并发语义。ext2 完整对齐 storage block 必须直接读取到 caller buffer，只有 partial block 可使用 scratch。`page_cache::RegularFile` concrete facade 只允许 syscall regular I/O 与 task ELF loader 持有：syscall 每次调用解析一次并跨 user-copy chunks 复用，executable source 在完整解析生命周期复用；禁止恢复按 chunk/probe 传 inode 的 free-function 双轨入口。所有 regular write 必须先取得 `RegularFileWrite` 的 write_sequence gate，完整 scalar/vector syscall 只取得一次，并固定 write_sequence → operation 锁序；truncate、allocate、fd/global sync 使用同一外层顺序。page fault 与 AddressSpace 已持 mm lock 的 mapping sync 只能取得 operation，禁止获取 write_sequence，否则同文件 mmap user-copy 会自死锁或形成 mm → sequence / sequence → mm 反向锁序。truncate 提交 storage 与 page-cache 删除后必须经 stable weak slots 逐个撤销 EOF 外 PTE，整个 post-commit invalidation 不得分配、返回错误或持 registry lock 进入 AddressSpace callback；否则 snapshot OOM 会留下无法回滚的 stale mapping。`syscall::fs::io::regular::{read_vectors,write_vectors}` 是 scalar、vector 与 positioned handler 共用的唯一 chunk/user-copy/partial/RLIMIT engine；其 staging 固定为 PAGE_SIZE，与 page-cache 和 user-copy 页粒度一致，禁止恢复会令每页重复多次 facade/page-walk 的小块 policy。ordinary vector operation 必须整次持有一个 OFD offset lock，禁止递归调用 scalar syscall 在 iovec 间释放 ownership。ext2 只在同一 mutation/JBD2 owner 内分配 hole、更新 block pointers/i_blocks/i_size。write/append 提交 storage 后只能获取一次 page-index lock，并按 `FallibleMap` ordered range 只访问写区间内实际 resident page；禁止按输入页扫描 cache hole、重复取锁或 clone Arc。`page_cache::writeback::writeback_range` 的 `pub(super)` 只允许 parent 的 mapping/fsync/sync orchestration 调用；它必须在 operation domain 内只取得一次稳定 EOF，以固定 32-entry scan batch 只 clone dirty Arc，并复用唯一 PAGE_SIZE stack scratch。`reclaim_under_pressure` 只允许 `MemoryReclaimer for CachedFile` 调用：必须 try-lock write-sequence/operation，单轮最多选择 32 个零 writer、无外部引用的 dirty 页；`Inode::try_write_storage_batch` 只允许该路径消费，adapter 必须在 filesystem mutation owner 忙时零副作用返回 `Busy`，禁止等待形成 allocator→filesystem 反向锁序。只有成功 transaction prefix 才可清 dirty，并仅在同一 Arc 仍由 cache+batch 独占时删除；Busy、I/O、OOM 与并发 writer/read 必须保留数据与 owner。嵌套 `writeback_batch::commit_with_backoff` 只允许 writeback parent 和 `tools/kernel-unit` production-path test 调用：完整 batch 先尝试一次 filesystem commit，普通 writeback 只以 `NoSpace`、pressure writeback 另以 durable publication 前的 `OutOfMemory` 作为可二分 capacity error；每个成功 slice 才发布 clean，后续 error 保留 suffix dirty。禁止按 range 构造 Vec、按页开始 journal transaction、按页读取 metadata 或分配临时 scratch，否则大 mapping munmap 会在内存压力下 OOM，且 32 页重新放大成 128 次 FLUSH。CachedPage 的 dirty bit 与 writable-PTE count 必须由同一 atomic state 拥有，writable publication 必须原子设置 dirty+writer，零 writer 与 clear-dirty 只能在一次 CAS 中提交；拆成两个 Atomic 会吞掉并发 writer publication。`SharedPage` 不得恢复无 writer lifetime 的 `mark_dirty` 入口，否则 writeback 无法用同一 state 证明 clean transition。大 range 允许拆成有界且各自完整提交的 transaction，禁止 syscall 写零覆盖已有内容、伪造成功或维护第二份 allocation map。
- BSD `flock` state 只由 VFS 以 mounted inode identity 持有，OFD pointer 只作为 open-file-description lifetime identity；Task wait registry 只拥有 interruptible membership，VFS notifier 只投递发生变化的 key。唯一 VFS lock-table mutex 覆盖 shared/exclusive 转换，释放后才调用 notifier；缺失该锁会让两个 exclusive holder 同时成功，反向持锁或在锁内通知会与 wait publication 死锁。最后 descriptor close 必须经 OFD descriptor_refs 释放，禁止按 Process/fd 复制 lock state或让 ext2 维护第二套 advisory lock。
- POSIX record lock 只由 VFS 以 mounted inode identity + Process TGID 持有；同 owner 的相交 range 在一次锁内拆分、替换、排序并合并。`F_SETLKW` 只把 membership 交给统一 indexed wait owner，VFS 锁外通知；任一该 inode descriptor close、CLOEXEC close 或 Process exit 必须释放 Linux process-associated locks，禁止按 fd/OFD/ext2 复制 lock table。
- `memory::reclaim_pages` 的 scoped caller 只允许 frame allocator 慢路径与 page-cache dirty throttle；后者必须在发布新的 writable shared PTE 前、dirty page 达物理页容量四分之一后按每新增 64 页的固定 cadence 请求 64 页，实际 resident scan 仍受 `ReclaimRequest` 约束。`page_cache::DIRTY_PAGES` 是 `CachedPage.state` clean/dirty 转换的只读计数投影：clean→dirty 恰好加一，CAS clean 或 dirty owner 最终 Drop 恰好减一；它不得成为第二份 page 状态或替代逐页统计。`DIRTY_THROTTLE_ACTIVE` 只合并跨 hart 的同一高水位尝试，成功取得者必须在有界 reclaim 返回后释放，未取得者由后续 cadence 重试；缺少该 gate 会重复扫描 owner/提交 FLUSH。该 throttle 只回写零 writer owner，不得等待仍可写 mapping 或在低内存后才首次申请 journal staging。
- Process exit 只负责关闭本 Process 的 descriptor/OFD lifecycle；禁止在 exit/close cleanup 隐式执行全局 filesystem sync。durability 只能经 ext2 journal/writeback owner 与显式 `fsync/sync` seam 提交，否则任意短命进程都能阻塞全系统 I/O，并把 close 语义错误扩大为全局持久化屏障。
- procfs 与 `sysinfo` 必须消费 task façade 的同一采集边界；`/proc/stat` 的 `btime` 只投影 timer 唯一拥有的 boot epoch。syscall 只编码 Linux UAPI，禁止解析 `/proc` 文本、复制统计状态或在 ABI 层维护第二套 uptime/load/memory/task counter。
- `uname` 只投影 system module 的 immutable identity；`getcpu/riscv_hwprobe` 只通过 system façade 投影 HartTopology 的 logical CPU/platform facts；`sched_getaffinity` 只读 SchedulingState owner；`gettimeofday` 与 `clock_gettime(CLOCK_REALTIME)` 只投影 timer realtime owner。禁止 syscall module 维护 hostname、ISA、CPU mask、release、timezone 或第二份 wallclock offset。
- MMIO/volatile 只存在于 arch/driver HAL；user pointer 只通过 AddressSpace copy；磁盘 packed layout 只存在于 filesystem adapter。
- syscall memory handler 只解析 Linux flags/prot/errno；TaskControlBlock/AddressSpace 只持锁转发；VMA 选址、冲突、split/merge、frame rollback 与 PTE 提交只存在于 MemorySet。
- Process resource-limit owner 只在 task layer 执行权限、fork/exec lifecycle 与 CPU/NPROC policy；syscall 只编解码 `rlimit64`，memory/fs 只消费数值上限，不得反向读取 Process。
- `task_manager::vfork::ProcessCloneError` 只穿过 fork/vfork façade 到 clone syscall：`Memory(OutOfMemory)` 映射 `ENOMEM`，PID/RLIMIT capacity 的 `ResourceLimit` 映射 `EAGAIN`，其他 memory validation 维持既有 `EAGAIN`；Thread clone 复用自身同义 `ResourceLimit`。任何 capacity failure 都必须发生在 graph/runqueue/TID copyout publication 前。
- frame allocator 的唯一慢路径通过 weak reclaimer seam 同步回收 `MADV_FREE`、clean private file/ELF 与无外部引用的 clean page-cache 页；private/shared dirty 页无 writeback 证明时禁止回收。`ReclaimRequest/Result` 同时约束目标页数与 resident-entry scan budget；registry 每轮最多访问 64 个 stable weak slot，并从同锁轮转 cursor 继续，adapter callback 前必须释放 registry lock。page-cache 与 private VMA 各自保存 owner-local page/VPN cursor，单次扫描不得越过 request，禁止从 map 起点重复 `find/extract_if`、构造临时容器或在足额后继续扫描。private reclaim 的 TLB flush 由实际 PTE 撤销决定，不能用物理 frame 是否最终释放替代；否则 COW frame 仍被其他 mm 引用时会留下 stale translation。kernel heap 只从 frame allocator 转移页，不维护第二份物理容量 owner。
- `memory::frame_allocator` 的 external interface 只包含初始化、默认 `Reclaimable` 单页分配、显式 `FrameAllocationClass` 的连续分配、统计与 `FrameTracker` lifetime；具体 buddy mutation 和错误类型保持 private。分配形状与 progress 等级正交：所有 page-table 与 anonymous/private/shared resident 消费者都必须使用 `Reclaimable` 并遵守一个最小 heap-growth extent 的低水位；普通 global-allocator 扩容只能使用 `KernelHeap` 并继续保留最终 cleanup reserve；只有启动期 DMA 可声明 `KernelCritical` 越过最终水位。禁止再以“连续分配”隐式取得 reserve。allocator 以一字节每页 block-state、每-order intrusive 双链 free list、nonempty bitmap 和同锁 free-block/free-page projection 组成唯一 buddy metadata；连续请求向上取整到自身对齐的 2ⁿ 页，order 选择 O(1)，split/merge 最多 O(usize::BITS)，single 常见路径 O(1)，不存在按 free pages 扫描。`FrameTracker` Drop 必须先验证起点/order owner state，再在同一次 IRQ-safe lock 内逐 order 合并；禁止逐页写 free links、逐页抢锁或恢复第二个容量 owner。
- global allocator 的 external seam 只有 Rust `GlobalAlloc`、初始化、只读统计与 frame-backed 切换；实现不泄漏 slab/direct adapter。8–1024 byte canonical class 使用单页 slab，每-class non-full 双链与 block free-chain 由唯一 IRQ-safe heap lock 拥有，alloc/free/list transition 均为 O(1)；free-chain 初始化必须在候选 `FrameTracker` 独占期锁外完成。更大或高对齐对象独占 order-aligned direct extent，header 与原始 `Layout` 必须唯一重建起点/order。slab empty/direct free 必须先撤销 heap membership，再在 heap lock 外用 `FrameTracker` 返还物理页；禁止跨 extent 合并、永久吞并 frame 或复制物理容量 owner。8–256 byte class 另使用动态 hart topology 等长的 per-hart cache；本地 pop/push 只关闭本 hart IRQ，不取 global heap lock，miss 在一次 slab lock 内最多补充 8 block，每 class 最多保留 32 block，overflow 按 canonical layout 归还 slab。cache cell 禁止按 raw hart ID、固定 hart 数或 task CPU 保存；cross-hart free 可迁移到 executing hart。frame allocation、direct reclaim、extent zeroing 与 frame return 都必须在 heap lock/IRQ-off 临界区外执行。
- futex private key 由 AddressSpace identity + uaddr 构成；anonymous/file shared key 只能由 MemorySet 从 backing identity + offset 归一化。IndexedWaitQueue 锁在 AddressSpace 锁外覆盖 key/value 比较、membership 发布、bitset wake 与 requeue，禁止 syscall、Process 或 scheduler 复制映射判断。
- task loader 是 pathname、Linux script rewrite 与 inode 到 `ExecutableSource` adapter 的唯一 owner；memory 只消费最终 ELF 随机读 seam，并唯一拥有 ELF 解析计划、PT_LOAD 映射、initial stack 与失败回滚。禁止恢复完整文件 `Vec`、filesystem 到 memory 的具体类型泄漏或第二套 script/ELF loader。
- thread-directed signal 发布到 Thread pending；kill/TTY/SIGCHLD 发布到 ProcessSignalState shared pending。两者经同一 delivery/wait seam 先发布 pending bit，再从 wait 的唯一 owner 注销 membership；blocking path 必须在 owner lock 内复查两类 pending，禁止 signal-before-enqueue lost wakeup。
- `task_manager::signal::select_and_generate_process_signal` 是 multi-process kill selector 的唯一 graph transaction：每个候选只在一次 process-graph lock 内完成 ordered selection、live representative、credential/SIGCONT-session permission、signal-zero probe 或 process-pending generation，以及 stop/continue consequence freeze；Process exit 和 group/session mutation 共用该 lock，禁止把 TGID、permission 或 generation 拆成多次事务。函数只返回 detached TGID、eligible Thread Arc、queued bit 与 parent notification；`publish_job_notification`、signal-waiter cancellation、interrupt 与 reschedule 必须在 graph guard 释放后执行。`task_manager::signal::selection_result` 只允许 parent signal loop 和 `tools/scheduler-unit` path test 使用，零分配 fold 只保存 successful count 与 denied fact；任一 probe/generation 成功后，后续 denial/missing candidate 不得把 syscall 结果改回 `EPERM/ESRCH`，全 denied 才返回 `EPERM`，空 selector 返回 `ESRCH`。禁止恢复 TGID snapshot、pre-generation success count 或兼容的多锁 generation path。
- stop/continue 冲突消除必须与 signal queue generation 同锁；`task_manager::signal::job_control` 封装 graph 内的 group-stop/continue、scheduler membership transition 与 parent notification，状态仍只由 Process graph 拥有，scheduler 只维护每个 Thread 的 `StopPending/Stopped` membership。禁止用第二套 stopped flag 与 run state 人工同步。
- Process graph 在 exec point-of-no-return 唯一发布 `has_execed`，parent-side `setpgid` 必须在同一 graph lock 下得到成功或 `EACCES`。Process 退出/重父子关系前后只在该 owner 内计算 orphan transition，并把需要的 HUP/CONT 记录为 graph-owned transient effect bits；释放 graph lock 后按全局稳定 cursor 零分配排空，先向所有目标发布 SIGHUP，再发布 SIGCONT。禁止冻结可因重父子变化失效的 TGID snapshot，或在 syscall、Terminal、scheduler 复制 parent/SID/PGID/orphan 状态。
- global init 的 unkillable policy 位于统一 signal generation/delivery：默认 disposition signal 被丢弃，显式 handler 和 blocked pending 仍遵循 Linux 语义，强制同步 fault 不经该豁免。默认 SIGTSTP/SIGTTIN/SIGTTOU 在 delivery 时必须复查 orphan group，SIGSTOP 不豁免。
- Terminal 唯一判断 controlling session、foreground group 与 `TOSTOP`；TaskManager 唯一判断 caller process group、orphan 状态及 signal mask/disposition。后台访问只经该 seam 产生 SIGTTIN/SIGTTOU、EIO 或 syscall restart，禁止 syscall pathname/device 特判复制 job-control policy。
- session leader 退出使用固定 graph -> Terminal 锁序原子取走 controlling session/foreground PGID，并在 graph lock 内冻结 foreground target；SIGHUP 必须在锁外经统一 signal seam 发布，禁止按可能已复用的 PGID 重新选择目标。
- UART hardirq 不调度、不分配，只清空设备 FIFO并发布 console softirq；console read 在统一 indexed wait owner 内复查 RX ring，deferred consumer 才移除 membership 并 wake task。
- syscall handler 只能向 dispatcher 返回内部 restart 结果；trap layer 将其暂存为 `EINTR` 并把原 `a0..a5/a7/ecall PC` 交给当前 Thread。实际交付的 handler disposition 含 `SA_RESTART` 时才把 replay context 写入 signal frame，否则 frame 保留 `EINTR`；内部结果不得进入 U-mode。
- dispatcher 对未接入 number 只返回 `ENOSYS`，不得为每次未知调用打印日志；syscall number 与调用频率完全由 U-mode 控制，逐调用日志会把正常 capability/io_uring feature probe 变成 console I/O 放大与拒绝服务面。诊断必须使用显式临时探针并在完成后移除。
- Thread exit 发布顺序固定为 robust cleanup -> process graph removal -> clear-child-tid/futex wake；join completion 不得早于 Thread owner 注销。exec commit 必须在替换旧 AddressSpace 前执行同一 robust cleanup 并清 registration。`model::robust_list::set_robust_list` 只允许 syscall registration 调用，长度正确的 NULL head 注销，`cleanup_robust_list` 只允许 `model::process_exec` 与 `task_manager::process_exit` 的 calling-Thread lifecycle 调用；模块持有 Thread robust-list registration 的唯一 mutation seam，不另存 membership。cleanup 必须一次快照旧 AddressSpace 与 user-fault limits，禁止在 CAS 持 mm lock 时再取 resource-limit lock；`task_manager::futex_wake_with_key` 只允许普通 futex wake 与 robust old-mm resolver 调用，固定 queue→memory 锁序，退出 MemorySet/queue guard 后才 wake task。non-PI robust OWNER_DIED publication 必须在每次 CAS conflict 后用返回的 live word 重查 owner TID，无独立 retry cap；成功替换且旧值含 WAITERS 才 wake。`list_op_pending` 的 zero-owner 状态必须补偿 userspace 尚未来得及执行的 wake，pending node 若已在主链只能处理一次。NULL/unaligned address、link read、futex read/CAS fault 必须停止 traversal；第 2048 个已处理 node 的 next 超出 traversal bound，不得阻止最终 pending cleanup。
- vfork caller 使用独立 `WaitMembership::Vfork(child)` 且不可被 signal 提前解除；child Process 与 parent 精确共享同一 AddressSpace Arc，并使用按 TID 分配的 supervisor trap page。只有 exec 原子替换 child handle并删除临时页，或 exit 先删除临时页，才能消费 child node 的唯一 caller waiter；不得挂起整个 parent Process。
- child exit/stop/continue event 必须在 process graph 锁内 claim；status copyout 成功后才消费，失败必须 release。多个 parent Thread 可分别等待，但同一 event 不得被重复返回或回收。
- private expedited membarrier registration 只存在于 AddressSpace；syscall 只能通过 task façade 请求 arch 对所有 active DTB hart 做同步 full fence。不得用本地 fence、RFENCE、吞 `ENOSYS` 或 musl fallback 冒充跨 hart memory ordering。
- terminal exit 必须先在可返回的 prepare frame 完成副作用并释放全部 task Arc，再以只含 raw context 的凭据切到 idle，由 per-hart deferred-reap slot 保活并析构 TCB；禁止从持有 task Arc 的 Rust frame 永久切走。
- user trap return 的 noreturn trampoline 前必须显式释放当前 TCB Arc；该 Rust frame 不会展开，依赖作用域析构会让每次 syscall 永久增加一个 kernel-stack 自引用。
- 首个 `exit_group` 或默认致命 signal 在 process graph 唯一提交 group-exit status；所有 sibling 只经 signal/wait/scheduler seam 回到自身内核栈退出，最后一个 Thread 才发布 zombie 与 SIGCHLD。禁止远程释放运行栈、重复保存 status 或把 signal death 改写成 shell exit code。
- raw CSR、DMA、page-table pointer、trap context 和 packed disk unsafe 必须有局部 `SAFETY:` 证明。
- 禁止 `static mut`、私有 syscall、固定 hart 容量、console syscall 旁路、deprecated/feature-flag 双轨。
- 禁止 `common/utils/helpers/misc/manager/base/shared/core` 等无领域含义的目录。
- `user/` 顶层只允许固定 BusyBox config、passwd/group identity、inittab/network service/udhcpc lease script、musl consumer、dynamic-loader C probe 与单 ELF `liteos-stress`；后者只通过 `cputest/memtest/cachetest` hardlink 提供有界 CPU、anonymous/COW 和 shared-file dirty/reclaim 诊断，不得形成第二 runtime 或默认 init。围栏禁止 Rust user crate/source/linker、`build-user` 和旧 init artifact。

## 6. Change contract

修改前必须确定所属 module、状态 owner、interface 变化、依赖变化以及 error/exit/interrupt cleanup。依赖采用正向 allowlist，未列出的跨 module 依赖一律失败。扩大 interface 或新增 global state 必须更新对应权威契约；不得修改围栏来掩盖实现问题。只有其他架构规则全部通过时，`cargo run -p architecture-check -- --write-interface` 才能更新 interface contract。

唯一强制验证入口是 `make verify`。静态围栏、build 与 artifact gate 成功后，topology、musl、BusyBox 与 APK runtime gate 才可并发；每条 gate 必须从 `target/rootfs.img` 派生独占可写镜像并使用独立 success stamp，禁止读取或改写开发 `fs.img`；网络 fixture 必须使用不相交的内部 host port domain。recursive Make 等待全部 gate，任一失败都禁止进入最终 diff hygiene；禁止以并发为由跳过、抽样或提前发布成功证据。
