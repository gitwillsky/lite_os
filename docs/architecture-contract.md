# LiteOS Architecture Contract

本文只定义稳定的 module interface、依赖和状态 owner。当前功能事实属于 `architecture.md`，Linux ABI 状态属于 `syscall-support.md`。

## 1. Crate contract

- `bootloader` 是独立 M-mode domain，不依赖 kernel 或 userspace。
- `syscall-abi` 只保存 kernel dispatcher 接入的 Linux/riscv64 ABI 常量，不依赖实现 crate。
- 产品 userspace 只有固定上游 musl + BusyBox rootfs；禁止恢复 Rust user crate、自有 runtime/init 或第二条默认镜像路径。
- kernel 的 `main.rs` 是唯一 composition root；初始化顺序和 adapter 装配不得下沉到 driver、filesystem 或 task。

## 2. Kernel dependency contract

| Module | 允许依赖（机器读取） | 说明 |
|---|---|---|
| `arch` | `config`, `memory`, `sync` | architecture mechanism 不得消费上层领域状态 |
| `config` | 无 | 只保存无运行时依赖的常量 |
| `sync` | `arch` | 只依赖本地中断机制 |
| `memory` | `arch`, `config`, `id`, `random`, `sync` | 不感知 task、filesystem 或具体 driver policy |
| `drivers` | `arch`, `memory`, `sync` | 不感知 task、filesystem 或 syscall |
| `ipc` | `id`, `sync` | 只拥有 Pipe byte/endpoint，不感知 fd、task、socket 或 syscall；`id` 仅分配 anonymous inode identity |
| `socket` | `drivers`, `id`, `ipc`, `sync`, `timer` | 拥有 socket domain facade、AF_UNIX 与 AF_INET stack；`drivers` 只允许 network-device seam，`id` 仅分配 anonymous inode identity |
| `fs` | `drivers`, `ipc`, `memory`, `socket`, `sync`, `timer` | `drivers` 仅允许 `block` seam；socket 仅允许统一 OFD backend facade；`memory` 仅允许 shared-page seam |
| `task` | `arch`, `fs`, `ipc`, `memory`, `socket`, `sync`, `timer` | 不依赖具体 device 或 syscall/trap entry；只在 deferred context 推进 network stack |
| `trap` | `arch`, `drivers`, `memory`, `syscall`, `task`, `timer` | 只做入口、分类和事件投递 |
| `syscall` | `fs`, `ipc`, `memory`, `random`, `socket`, `system`, `task`, `timer` | 不得绕过 facade 接触 adapter/scheduler/page table |
| `random` | `drivers` | entropy facade；只消费 RNG device seam，不生成伪随机 fallback |
| `system` | `arch` | 只拥有 whole-system reset/shutdown/CAD policy |
| `timer` | `arch`, `config`, `drivers`, `sync` | RTC adapter 由 timer 唯一拥有 |
| `log` | `arch`, `sync` | 日志策略和输出在本 module 内闭合 |
| `id` | 无 | 纯 ID allocation mechanism |
| `lang_item` | `arch` | 只使用 architecture fail-stop mechanism |
| `main` | `arch`, `config`, `drivers`, `fs`, `id`, `ipc`, `lang_item`, `log`, `memory`, `random`, `socket`, `sync`, `syscall`, `system`, `task`, `timer`, `trap` | 唯一 composition root |

同一 module 内引用不构成跨 seam 依赖。`main.rs` 可以依赖所有 kernel module，但只能做装配、启动顺序和 fail-stop 策略。

## 3. State owners

| 状态 | 唯一 owner |
|---|---|
| M-mode initialization、HSM、RFENCE | bootloader 对应 module |
| DTB board facts | immutable BoardInfo publication |
| raw hart possible/online/active、startup stack、同步 memory-barrier request/completion generation | HartTopology；有序 hart table 与同批发布的 immutable raw-ID → compact-index projection 唯一定义紧凑 index，每次调用的 generation 由 arch hart barrier mechanism 唯一分配 |
| per-hart current、runqueue、mailbox | task ProcessorTopology |
| per-hart startup stack、softirq pending、timer deadline、memory-barrier generation 与 online/active publication | arch HartTopology；timer 只经 `advance_timer_deadline` 推进所属 hart 的固定相位 deadline；timer backlog 使用独立合并 softirq bit，禁止从 handler 完成时刻另存下一 tick 或复制 pending state |
| task run state、generation、wait membership、wake result 与 per-Thread logical CPU affinity | SchedulingState；affinity 与 membership 同锁，禁止按 syscall/CPU 复制 mask |
| per-Thread nice/vruntime、active CPU slice/dispatch weight、累计 runtime、legacy scheduler policy/reset-on-fork 与 I/O priority | Sched policy lock；fork/clone 只经 `Sched::forked` 派生，exec 保留；ioprio syscall 不保存 ABI shadow state |
| process address-space handle、cwd opened-entry、fd table、credentials/umask、resource limits、聚合 CPU runtime | Process；Thread 共享，fork 复制 limits 并重置 runtime，exec 保留；vfork child Process 初始持同一 AddressSpace Arc，exec 只替换 child handle；最后一个 Thread exit 立即取走 fd table |
| VMA 与 private expedited membarrier registration | AddressSpace；CLONE_VM/vfork 共享，fork/exec 新 owner 从未注册开始 |
| PID/TID allocation、parent edge、live thread collection、process-creation reservation、exec generation、group-exit status、job-control/orphan lifecycle 与 child event/waiter | TaskManager process graph；独立 creation lock 只覆盖 `RLIMIT_NPROC` 检查到 graph publication，防止并发超限；vfork child node 唯一持有 suspended parent Thread |
| ITIMER_REAL record 与 active deadline index | TaskManager 唯一 RealTimerQueue；fork 不继承、exec 保留、最后一个 Thread exit 删除。set/get/exit 固定 graph → timer 锁序；timer deferred 只持 timer lock 摘取与重装 deadline，释放后才经统一 signal seam 发布 SIGALRM |
| deadline/futex/pipe/poll/signal/console wait registration、event filter、exclusive mode 及其 indexes | TaskManager 唯一 IndexedWaitQueue；futex index 只消费 memory 归一化的 private/shared key，requeue 原地迁移同一 registration；blocking pipe writer registration 保存本次原子写所需容量，poll 仍只观察通用 endpoint readiness；ppoll/epoll/socket blocking 共用 source indexes；socket 内部 edge notification 固定注册为 source-native read key，userspace event mask 只用于真实 data source 和后续 level recheck；source wake 唤醒全部普通 callback group（每个 epoll instance 一个 thread）及一个 exclusive group |
| signal disposition、process-directed shared pending set | Arc<Process> 的单一 ProcessSignalState lock |
| signal mask、thread-directed pending set、alternate stack registration、active frame | ThreadContext 与用户 RV64 rt_sigframe；altstack active 只由用户 SP/range 推导，不复制 flag |
| interrupted syscall 的单次 replay record | ThreadContext；signal frame 保存最终 replay/EINTR 上下文 |
| OFD backend、opened-entry identity、offset/status flags、跨 fd-table descriptor 引用数 | OpenFileDescription；pathname-backed character/regular/directory fd 保留 VFS opened entry，anonymous pipe 使用 pipefs 语义；最后 descriptor close 触发 epoll interest cleanup |
| anonymous Pipe/AF_UNIX stream 共用的 byte ring、endpoint count、pipe `PIPE_BUF` atomicity、stream short-write policy 与内核 readiness notification token/generation | ipc::Pipe；mode 由调用 seam 明确选择，不复制到 fd table、socket state 或 wait registry；notification token 只合并待消费 edge，每次 signal 仍推进 generation 并通知 registry，registry owner lock 内先排空 token、再做 socket level recheck |
| AF_UNIX abstract namespace、endpoint state、listen/datagram queue | socket::UnixSocket；OFD 只持统一 Socket Arc，fork/dup 共享，具体 domain adapter 不穿透 fs seam |
| Ethernet interface IPv4 address/prefix/default route、ARP cache、UDP/TCP/raw ICMP socket set、peer、IP_PKTINFO、socket option、ephemeral port、TCP listener backlog 与 FIN/TIME_WAIT orphan | socket::NetworkStack；ioctl、packet dispatch、procfs 只读同一 owner 快照；accepted handle 只转移给新 Socket/OFD，不复制协议状态 |
| AF_PACKET binding、protocol 与有界 receive queue | socket::PacketRegistry；RX frame 在 smoltcp ingress 前只镜像一次，packet endpoint 与 L3 NetworkStack 不复制彼此状态 |
| VirtIO-net RX/TX queue、DMA buffers 与 packet/byte counters | VirtIONetworkDevice；hardirq 只确认并发布 network softirq，deferred context 才解析协议与唤醒 Pipe waiter |
| epoll `(fd, OFD)` interest、ET generation、MOD revision、delivery cursor、ONESHOT state、ctl notification 与无环嵌套图 | fs::Epoll；内部 notification Pipe 与 readiness generation 均接入 ppoll 的同一 source/wait seam |
| eventfd 64-bit counter、semaphore mode 与 read/write readiness generation | ipc::EventFd；OFD/fd table 只持 Arc，notification Pipe 只承载 edge，不复制 counter |
| program break；VMA 区间、类型、权限、private dirty/discardable residency、anonymous shared backing 与 framed page lifetime | MemorySet 的精确 program-break 字段与有序 VMA 表；brk 页使用可拆分/替换的普通 anonymous VMA，ELF/anonymous/stack/file 页按 fault 驻留，anonymous shared backing 按索引唯一发布跨 fork frame，PageTable 只保存硬件 translation |
| physical frame lifetime | FrameTracker/frame allocator |
| process comm/创建时刻、thread 创建时刻、mm argument range/主 ELF code range/VMA statm、thread runtime 与 run state | Process、ThreadContext、MemorySet、SchedulingEntity；Process runtime 聚合保留 exited Thread，procfs 只读取 owner 快照或 mm 中的实时 argv bytes |
| per-Thread 与 Process 聚合 I/O counters | ThreadContext 与 Process 各自 IoAccounting；每次完成只推进当前 Thread 和所属 Process，Process owner 保留 exited Thread 历史；procfs 只读快照 |
| termios、cooked input、controlling session、foreground process group | Terminal；TaskManager 只读取 job-control 判定结果，不复制 TTY 状态 |
| per-hart busy runtime | ProcessorTopology 对应 raw hart slot；procfs 经 HartTopology index 投影 logical CPU，不另建 counter |
| 1/5/15 minute load average、5-second cadence claim 与 in-progress generation | TaskManager 唯一 LoadAverage；atomic deadline 只分配周期，零 sentinel 串行化跨周期采样，三个 FSHIFT=11 value 由单一 IRQ-safe lock 原子发布 |
| system information snapshot | task façade 只投影 allocator、process graph、load average 与 timer 的权威状态；procfs/sysinfo 不另建 counter |
| immutable system/build identity | system module；uname 只编码，不复制 hostname/release/machine state |
| realtime offset、boot epoch、timebase resolution、scheduler tick interval 与固定 UTC timezone policy | timer module；clock_gettime/gettimeofday、scheduler interval 和 task/procfs 投影共用同一 timer owner |
| root mount、source/filesystem association、boot-time mount table、mount enter/leave、pathname traversal 与 namespace mutation publication order | VFS；mutation lock 覆盖 adapter commit 到 opened-entry registry publication |
| opened-entry parent/name/deleted 关系与 live weak registry | VFS；cwd/OFD/procfs 只持 Arc 或读取投影，rename/unlink 只在 VFS 更新 |
| boot 内 anonymous Pipe/Socket object identity allocation | id module 分配机制；具体 object owner 持有，fd table/procfs 不复制 |
| inode/on-disk allocation、statfs capacity、JBD2 active transaction/recovery sequence 与 orphan chain | filesystem adapter mutation domain；ext2 统计、allocator、journal 与 orphan lifecycle 共用唯一 mutation 顺序 |
| regular inode cached pages、fill/mutation/writeback order 与跨 OFD write sequence | page_cache::CachedFile；内层 operation lock 保护单次 storage/cache commit，外层 write_sequence gate 保护完整 syscall write/truncate/allocate/fd-global sync，page fault 与 AddressSpace-owned mapping sync 禁止获取外层 gate |
| page-cache resident/dirty/reclaimable 统计 | page_cache::CachedFile 的唯一 page maps 与 CachedPage state；procfs 在读取边界汇总，不保存 shadow counter |
| VirtIO descriptor/DMA lifetime | VirtQueue/driver instance |
| entropy device 与请求串行化 | VirtIORngDevice；random facade 不缓存或派生第二份状态 |
| UART MMIO 与固定容量 RX ring | UART driver；hardirq 只填 ring，console waiter 只由 deferred softirq 消费 |
| interrupt registration/affinity | interrupt controller |
| syscall number | syscall-abi |
| mounted inode 的 BSD advisory flock state | VFS；key 为 filesystem+inode，holder 为 OFD identity |
| mounted inode 的 POSIX byte-range record-lock state | VFS；key 为 filesystem+inode，holder 为 Process TGID；TaskManager 只拥有 interruptible wait membership |
| user pointer与 errno translation | syscall module |

新增 global、Atomic、lock、cache 或 flag 必须在声明附近写 `OWNER:`，并说明缺失该 owner 会造成的具体状态分裂。

## 4. Source size contract

生产 Rust 源文件采用两级围栏：超过 600 行触发 architecture review notice，但不单独导致验证失败；超过 1200 行默认拒绝。reviewer 必须检查 owner、依赖方向、公开接口与真实领域 seam，选择拆成深 module，或在下表登记精确审查额度。登记是超过 1200 行的唯一例外入口，也可用于记录 601–1200 行文件的审查结论；必须同时给出状态 owner、不可立即拆分的原因与消除条件。每个登记额度就是该文件的硬上限，只能随重构下降，不得为功能开发上调；文件低于登记额度时 checker 强制同步下调。行数只是退化信号，禁止按行数机械切片或建立 pass-through module。

| Source | Reviewed max lines | Owner | Reason | Exit criterion |
|---|---:|---|---|---|
| `kernel/src/fs/ext2.rs` | 2288 | `fs::ext2` | ext2 inode、allocator 与 packed layout 仍共享同一 mutation domain；storage mutation 已下沉 | 提取不泄漏 packed layout 的 inode/allocator 深 module 后下调额度 |
| `kernel/src/memory/mm.rs` | 889 | `memory::MemorySet` | 核心 VMA 表示、页表提交、program break 与 kernel/system mapping 仍共享底层 PageTable/frame 不变量；ELF load、heap 专用 VMA mutation、mapping request、mmap、user-copy、shared/private area、statistics 与 futex key lifecycle 已下沉 | 提取不泄漏 PageTable/frame 的 kernel mapping 与剩余 VMA mutation 深 module 后下调额度 |
| `kernel/src/task/model.rs` | 603 | `task::model` | Process/Thread 核心 owner、constructor 与共享资源 handle 仍共同约束 clone/fork/exec 生命周期；credentials、address-space、scheduling、signal、I/O accounting 与 resource limits 已下沉 | 将剩余 ThreadContext lifecycle 按不泄漏 TCB 内部字段的深接口下沉后删除该额度 |
| `kernel/src/fs/file.rs` | 606 | `fs::OpenFileDescription/FileDescriptorTable` | OFD kind dispatch 与 descriptor table 仍共享 close/dup/readiness 生命周期；terminal/proc projection 已下沉，当前再拆会建立 pass-through dispatch | 当 anonymous descriptor kind 可由统一 backend trait 封闭 readiness/stat/proc projection 时下沉并删除中央枚举分派 |

## 5. Interface and capability contract

- kernel 与 bootloader 是 binary crate，跨 module interface 使用最窄的 `pub(super)`、`pub(in …)` 或 `pub(crate)`；不得使用裸 `pub` 伪造外部 interface。
- `memory::mm::mmap` 拥有 `PageFaultAccess/Outcome` 及其 retry postcondition；`memory::mm::user_access` 唯一拥有完整 range fault-in、权限证明与 copy。`prepare_user_read` 只向 sibling `futex_key` 开放，使 key 归一化前取得同一 `U|R` 证明；write prepare 保持 private，因为 copyout/CAS 是唯一调用者。prepare 必须以已命中的 `user_page` 或 `PageFaultOutcome::Handled` 完成逐页权限证明，禁止恢复独立 `validate_user_range` 或 fault 后重复翻译；额外 validation pass 会让所有 user-copy 对每个页重复一次 Sv39 page-table walk，却不增加失败原子性。program break 由 `MemorySet` 单字段拥有，实际页复用 anonymous VMA；若恢复特殊 heap VMA，`MAP_FIXED`/`mprotect` 会绕过统一拆分语义。`mmap/mprotect` 必须接受 Linux `PROT_*` 合法组合（包括显式 W+X），ELF segment policy 留在 loader；在通用 VMA/PTE 层强制 W^X 会把 V8 JIT 的合法请求误报为 `EACCES`。`MAP_NORESERVE` 只在 syscall ABI seam 消费；无 swap/commit reservation owner 时不得向 VMA 增加 shadow flag，拒绝该标准 advisory flag 会让 V8 的 PROT_NONE reservation 在进入唯一 lazy mapping owner 前失败。
- `model::alternate_signal_stack` 是 syscall、signal delivery 与 sigreturn 共用的唯一 Thread altstack seam：syscall 只编解码 RV64 `stack_t`/errno；module 内部唯一执行 `SS_ONSTACK` 动态投影、RISC-V frame placement、nested overflow、`SS_AUTODISARM` commit/restore 与 fork/clone/exec lifecycle。禁止保存第二个 active flag，禁止 signal_state 自行计算 stack top；前者会与用户 SP 分裂，后者会绕过 overflow 和 autodisarm commit 时点。
- `processor::job_control::request_reschedule_on` 只向 parent scheduler 的 raw-target control path 开放；它映射一次目标并复用 processor 私有的 `publish_reschedule_at` 提交点。Ready delivery 已持有 verified compact index 时直接复用同一提交点；两条路径都必须统一发布 per-hart flag，并在远端用同一次 SBI IPI 唤醒 mailbox，其他 module 不得直接调用。`request_tick_reschedule` 只向 task deferred timer consumer 开放，并且只有本 hart 同时存在 Running 与 queued/inbound runnable membership 时才发布 reschedule；无条件 tick yield 会让单任务每 tick 自我切换。signal publication 若未消费 wait membership，必须显式请求 Running target reschedule，禁止再依赖 timer 的偶然抢占。
- `model::scheduling::Sched::new` 只向 Process/Thread constructor 开放；`begin_runtime` 只由 `switch_to_task` 开始 active CPU slice，`checkpoint_runtime` 只由本 hart timer deferred consumer 提交已消耗增量而不结束 slice，`finish_runtime` 只由当前 task 的 block/preempt/vfork/exit orchestration 提交剩余增量并结束 slice。checkpoint 必须同步推进 Thread/Process/hart 累计与 Q10 vruntime，并保留 dispatch nice 快照；若只刷新 per-hart counter，会复制 runtime owner并在后续 finish 重复计费。`TaskControlBlock::cpu_runtime_snapshot` 只供 `clock_gettime` 与 same-thread-group procfs 只读投影 checkpoint 后的 active delta，不提交 runtime；`thread_statistics` 只向 task façade 的 procfs 采集开放，calling Thread 包含 active delta，其他 hart 上 running Thread 保持最多一个 scheduler tick 的 bounded-stale 累计值。Process 聚合 runtime 仍只读取其唯一 owner。nice scaling 固定使用 Linux v7.1 的 40 项 reciprocal 表并在热路径以乘法/移位推进，禁止复制权重表或退化为分档/运行时除法。重复 finish/checkpoint 必须 fail-stop，snapshot 不得写回或累加 live Thread 私有值，否则同一时间片会被重复观察，且 exited Thread 的 Process runtime 会丢失。
- `timer::monotonic_resolution_ns` 只向 `clock_getres` 开放，直接从 immutable DTB timebase frequency 换算且不缓存第二份 resolution；CPU clock resolution 由 scheduling 的微秒记账单位决定。`scheduler_quantum_ns` 只向 TaskManager scheduler interval seam 开放，从 timer owner 已校准的真实 tick interval 换算，syscall 不得复制 `TICKS_PER_SEC`。`arch::hart::advance_timer_deadline` 只向 timer interrupt programming 开放，并在内部选择 calling HartState；所属 hart 从上一 deadline 跳过已错过周期，避免 handler 延迟累积成相位漂移，timer caller 不得取得具体 state adapter。
- `timer::realtime_deadline_to_monotonic_ns` 只向 absolute realtime `clock_nanosleep` 开放，使用 timer owner 的 immutable boot offset 做单次精确域转换；syscall 不得先后采样两只时钟再估算，否则采样间隔会被错误加入 deadline。
- `task_manager::sleep_until` 只接受 absolute monotonic deadline 并返回领域 `WaitResult`；`nanosleep/clock_nanosleep` syscall 唯一负责 clock selector、relative/realtime 换算、remaining copyout 与 errno。task 层不得返回 ABI errno 或解释 userspace timespec，否则两个入口会形成独立 deadline/restart 语义。
- `task_manager::context_switch::prepare_current_block` 是 Running → Blocking 的唯一 publication seam：caller 把覆盖 readiness/signal 复查的 registry、process graph 或 unit owner guard 整体移交，closure 只发布对应 waiter 并返回 `WaitMembership`；seam 在返回 `PreparedBlock` 前提交 active runtime、取走 current、在 scheduling lock 下绑定 membership/state，并强制释放 owner guard。`PreparedBlock::suspend` 消费 token 一次完成 context switch 与 wake result 获取；未消费 token 的 Drop 必须 fail-stop，因为此时 current 已撤销且 task 会永久停在 Blocking。任何 caller 手写该序列都会重新引入 wait entry/state 分裂、runtime 重复提交或持 owner lock 切换导致的永久死锁。
- `task_manager::scheduler_affinity` 是 syscall 到 scheduler 的唯一 affinity seam：内部选择全局 live TID、执行 credentials permission、规范化 active mask、替换 Ready generation、请求 Running migration并同步等待 ownership 释放；syscall 只编解码变长 cpumask/errno。拆开会让 set/get、wake 与 migration 分别维护不同 mask，或在 syscall 成功后继续于禁止 CPU 执行。
- `task_manager::scheduler_policy` 是 legacy scheduler policy/param syscall 的唯一 seam：内部选择全局 live TID、保持 Linux target/validation/permission 顺序，并在 Sched policy lock 下查询或替换 reset-on-fork；相邻 `scheduler_rr_interval` seam 只完成同一 TID selection 后读取 timer owner 的固定 `SCHED_OTHER` quantum。syscall 只编解码 `sched_param`/`__kernel_timespec`、纯 priority bounds 与 errno。`thread_selector::scheduler_thread` 只供 policy/interval/affinity seam 复用零/current 与全局 TID 选择；`TaskControlBlock::scheduler_privilege_for` 也只向 policy/affinity 发布同一次 caller credential snapshot。`Sched::reset_on_fork` 只供 policy seam 查询/替换，`Sched::forked` 是 Process/Thread constructor 消费该 flag 的唯一入口；绕过会让 get policy 与 child nice 继承分裂。当前 setter 只接受 `SCHED_OTHER`/priority 0；没有独立 scheduling class 时不得开放 RT/BATCH/IDLE/DEADLINE/EXT setter，固定 interval 也不得宣称 Linux EEVDF dynamic entity slice 或 RT RR class。
- `task_manager::scheduler_nice` 是 get/setpriority 的唯一 collection seam：内部解释 PROCESS/PGRP/USER 零 selector、复制 live Thread owner、执行 effective-UID/root 与 target RLIMIT_NICE 判断，并保留 Linux 多目标部分成功及后续错误覆盖语义；syscall 只映射 selector、raw `20-nice` 与 errno。`Sched::nice` 只向该 seam 和 scheduler/procfs projection 开放；active slice 必须冻结 dispatch nice index，修改后经统一 reschedule seam 尽快切出，禁止按新权重追溯计费已运行时间。
- `task_manager::scheduler_io_priority` 是 `IOPRIO_WHO_PROCESS` query/set 的唯一 target、credential 与 policy mutation seam；syscall 只校验 encoded class/data 和映射 errno。`Sched::io_priority` 只向该 seam 开放，fork/clone 必须经 `Sched::forked` 继承，exec 保留；尚无 block scheduler enforcement 时不得声称 RT/BE/IDLE 改变实际 I/O dispatch。
- `arch::hart::hart_index` 是任意 raw hart ID 到紧凑 topology index 的唯一 O(1) projection；`current_hart_index` 用同一次 validated current-entry lookup 直接返回 calling index。其 fixed-width immutable rank table 只与有序 HartState table 一次性发布。ProcessorTopology 必须按同一顺序构造：current slot 与 local runqueue 操作直接使用 `current_hart_index`，选核扫描直接使用已有 candidate index，只有远端 raw target 才使用 `hart_index`，且一次映射必须同时定位 HartState 与 Processor slot；禁止再建映射、组合 `hart_id()+hart_index()`，或在 trap/scheduler 热路径扫描、二分 hart table。system façade 的 `getcpu/riscv_hwprobe`、SchedulingState 的 `CpuAffinity` 与 procfs CPU/processor 字段必须共用该顺序；SBI/PLIC/scheduler mailbox 继续使用 raw hart ID。`mark_online/mark_active/advance_timer_deadline` 由 arch façade 内部选择 calling HartState，具体 mutation method 保持 private。混用会在稀疏 DTB 上暴露不存在的 CPU number或把任务投递到错误 hart；分离可变 projection 会把 raw ID 路由到错误 per-hart owner。
- 默认 private；Rust AST 围栏解析所有 scoped visibility declaration、字段、方法、trait item 与 enum variant，连同可见域由 `architecture-interface.txt` 完整记录。
- `task_manager::wait_registry` 的 scoped interface 只允许 parent orchestration、`prepare_current_block` 的 publication closure 与 sibling signal cancellation 使用；它唯一执行 membership/index 的 insert/remove/take，caller 不直接修改 `entries` 或 source index。timer deferred owner 在一次 registry lock 内最多摘取 32 个到期 entry，再锁外完成 scheduling wake；仍到期的 backlog 必须发布合并 `TIMER_BACKLOG_SOFTIRQ` 续批。逐 entry 重新抢锁会放大关中断开销，无界 drain 会饿死 I/O/user-return，等待下一 hardware tick 则会人为增加最多一个 quantum 的 timeout 延迟。
- `task_manager::RealTimerQueue` 的 scoped interface 只允许 TaskManager 构造和 Process exit lifecycle 使用；用户 set/get 由同 module façade 在 process graph live-TGID guard 下完成。queue 在同一 timer lock 内唯一维护 record 与 `(deadline, TGID)` index；deferred 每批最多摘取 32 个到期 timer 并沿原始相位重装，释放 timer lock 后才发送 signal，backlog 与 deadline wait 共用合并 `TIMER_BACKLOG_SOFTIRQ`。禁止从 process graph 扫描 timer、复制 active membership 或在 timer lock 内获取 graph；前两者会恢复每 tick 的 O(processes) 扫描与 heap allocation，后者会反转 graph → timer 锁序。
- `task_manager::LoadAverage` 的 scoped interface 只允许 TaskManager 构造、timer deferred 周期更新以及 procfs/sysinfo 读取。非到期 tick 只读取 atomic deadline，不获取 IRQ-safe value lock；到期窗口只能由一个 hart claim，零 sentinel 在 commit 前阻止后续窗口反序提交。采样只在 timer path 执行，读取不得触发 process graph 扫描。FSHIFT=11、5-second EXP constants、active 上升时向上舍入及 missed-period O(log n) power 必须保持 Linux v7.1 算法；当前 active scope 是所有 runnable Thread，尚无 Linux uninterruptible task state owner 时不得伪造对应贡献。
- `ipc::PipeWaitCondition` 只允许 syscall I/O 构造、`task_manager::pipe_wait` 消费；它把 `PIPE_BUF` 原子写的容量条件带入唯一 wait registration，poll key 不得复制这项 blocking-write policy。
- `ipc::PipeEnd::{signal_readiness,drain_readiness}` 只允许 epoll、eventfd 与 socket adapter 的内部 notification Pipe 使用；anonymous Pipe 和 AF_UNIX stream data Pipe 禁止调用。eventfd 的 Pipe 只发布 read/write edge，counter 仍由 `ipc::EventFd` 单锁唯一拥有；零值 write 不得制造虚假 readable edge。`socket::SocketWaitSource` 是 socket façade 到 syscall poll 的唯一 wait-source seam：`Notification` 固定映射 source-native read key，`Data` 只投影 AF_UNIX stream endpoint；backend 的 `wait_sources/consume_wait_notifications` 只向 parent socket module 开放。syscall blocking I/O 只经 `poll::wait_for_ofd` 执行 token drain、registry publication 与 level recheck，不得复制该序列。若后续用独立非 Pipe notification primitive 取代 token，该 primitive 必须接管同一 generation/notifier owner，并删除这些 Pipe 专用 scoped interface。
- `socket::UnixSocket` 只在 state lock 内选择/转移 endpoint；Pipe I/O、endpoint Drop 与 notification 必须在释放 state lock 后执行。AF_UNIX stream 必须使用 ipc owner 的 short-write mode，禁止复制 pipe atomicity 或在 socket 层维护第二份容量状态。
- filesystem 只能看到 `drivers::block` seam，不得看到 VirtIO adapter。
- ext2 只提供 persistent root；`/tmp`、`/root`、passwd/group 与 `/dev`、`/proc`、`/sys` mountpoint 都由唯一 rootfs builder 固化，禁止 kernel/syscall/applet 按路径补造。rootfs builder 只能把完整 tree 转成一个 signed `liteos-base` package，再由 target `apk.static` 创建最终 installed DB；官方 `ca-certificates-bundle` APK 唯一拥有系统 trust bundle，host 不得覆盖其路径。host 不得直接解包最终 runtime、伪造数据库或保留 package 外的同名文件 owner。本地 private key 只允许位于 ignored `target/apk-runtime`，镜像只安装 public trust root。builder 的唯一公开产物是可复现基准 `target/rootfs.img`；`fs.img` 只拥有开发期 guest mutation，缺失时初始化且不得因普通 build/run/verify 或基准 mtime 变化被覆盖，只有显式 `reset-rootfs` 可替换。运行时 devfs/procfs/sysfs 只经 VFS mount table发布；`/dev/fd` 与 stdio aliases 只由 devfs 指向 `/proc/self/fd`，禁止写入会被 mount 遮蔽的 ext2 节点。VFS 唯一保留 mount source 到 filesystem adapter 的关联，并向 procfs 发布 `/proc/mounts`；procfs 通过 `ProcSource` 反转依赖消费 task/memory/fd 快照，sysfs 只消费 composition root 传入的 immutable DTB logical CPU count，禁止 fs 反向依赖 task/arch、syscall pathname 特判或伪 regular-file 节点。
- cwd、directory fd 与 pathname-backed OFD 必须持有同一 VFS `OpenedFile` seam；`chdir` 只负责 pathname resolve，`fchdir` 只负责 fd/opened projection，二者必须共用最终 directory/search-permission 检查并且只替换 Process 唯一 cwd。VFS weak registry 唯一提交 rename/unlink 对 parent/name/deleted 的更新。禁止缓存绝对路径、由 inode 扫描猜测 hardlink 名称，或把 procfs ` (deleted)` target 当普通 pathname 重新解析。`/proc/<pid>/fd` magic link 可跟随 live opened entry，Pipe/Socket target 与 `fstat` 只投影其 object owner identity，eventpoll 使用标准 anon-inode label。
- `/proc/<pid>/fd` 只允许同 TGID、effective root，或 caller effective UID 与目标 real/effective/saved UID 全部相同的访问；credential 判定留在 task façade，procfs 不复制 Process credential state。未建模 fsuid/capability/dumpable 前不得扩大该边界。
- `/proc/<pid>/statm` 只投影 MemorySet 的一次 VMA/residency 快照；`/proc/<pid>/task/<tid>` membership 只投影 process graph 的 live Thread map，thread stat 的 TID、创建时刻、run state、nice、runtime 与 last CPU 分别读取 ThreadContext/SchedulingEntity owner。`ProcProcessSnapshot` 是 fs adapter 与 task façade 之间的唯一批量投影，禁止 procfs 按 TID 回调 task 或缓存 membership；否则一次 htop 扫描会退化为逐文件 graph lock，并让退出线程残留。当前没有 per-Thread comm mutation owner，thread `comm` 精确投影 Process 共享 comm，不复制不可维护的名称状态。
- `/proc/<pid>/io` 只读取 Process 聚合 owner，`/proc/<pid>/task/<tid>/io` 只读取 Thread owner；logical chars/syscalls 在 read/write operation 返回边界推进，storage bytes 由 regular cache miss fill 与同步 storage write 的实际成功前缀推进。禁止把 logical bytes 复制到 storage counters，或在 procfs 扫描 live threads 反推 Process 值，否则 cache hit 会冒充磁盘 I/O 且 exited worker 历史会倒退。
- `memory::mm::statistics::user_page_statistics` 只向 task AddressSpace 的 `page_statistics` 开放，后者再以 `ProcessStatistics` 只交给 task procfs façade；size/resident/shared/data 必须在同一次 mm lock 与 VMA scan 中取得，text 必须使用同一 MemorySet 在 exec transaction 中提交的主 ELF code range，禁止把 interpreter 或 JIT executable VMA 误计入，也禁止为 statm 建 shadow counter。`fs::procfs::{node,lookup,process}` 的 scoped interface 只由 parent procfs adapter 消费，分别封闭 inode identity、live snapshot lookup 与文本编码；不得把具体 ProcNode 穿过 VFS/ProcSource seam。
- `statfs/fstatfs` 只经 VFS/OFD seam 选择 filesystem；ext2 从同一 mutation domain 投影 superblock 容量，procfs/devfs/sysfs/anonymous pipe 使用 Linux simple-statfs 形状。禁止 syscall 识别具体 adapter、按 filesystem id 复制统计或伪造可分配容量。
- VFS 只决定 pathname、mount 与 cross-filesystem policy；ext2 的 create/link/unlink/rename、allocator、JBD2 write-set/commit/checkpoint 与 orphan recovery 必须在同一 mutation domain。禁止 syscall/VFS 复制 link count、journal 状态或用写序调整冒充跨块原子性。
- live mount root/mountpoint 不得被 unlink 或作为 rename source/replace target；VFS 在进入 adapter mutation 前返回 `EBUSY`，防止 mount table 与 opened-entry namespace 分裂。
- VFS permission evaluator 只消费 Process 发布的 immutable identity snapshot，唯一决定 traversal、inode rwx、parent mutation、sticky directory、protected hardlink 与 setgid-directory inheritance；syscall 和 filesystem adapter 不得复制权限 policy。ext2 只持久化 VFS 已决定的 mode/UID/GID/ctime。
- `syscall::fs::io::sequential` 是 `read/readv/write/writev` 的唯一 external seam；四个 wrapper 只完成 fd/access、iovec import 与 scalar-one-vector projection，所有 OFD kind dispatch 分别只存在于 internal `read::read_descriptor` / `write::write_descriptor` seam。恢复 scalar/vector 双轨会让 socket scalar I/O 再次受固定 staging 长度截断，并让 error/partial/signal 语义漂移。`user_vector::import_iovecs` 是所有 vector I/O 的唯一 import seam；连续 ABI array 必须按 PAGE_SIZE staging 批量 copyin，禁止恢复逐 iovec user-copy/page-walk。`UserIoCursor` 唯一维护 non-regular sequential scatter/gather 的 vector index、vector 内 offset 和 completed copy count，caller 禁止复制字段人工同步。socket datagram 的完整 scalar/vector 序列只能提交为一个消息；stream 可分 bounded staging，但首次 short/block/error 后必须返回真实 partial count。eventfd 的 scalar/vector read 都遵循 Linux `eventfd_read` 的统一 iterator contract：总 capacity 至少八字节并只 scatter 一个 u64；eventfd writev 保持 scalar write fallback，逐个非空 iovec 要求恰好八字节，禁止把跨 iovec prefix 合并成 value。
- `syscall::fs::io::sendfile` 的 regular→regular scope 必须只消费 page-cache `RegularFile` facade：输入/输出各解析一次，唯一 PAGE_SIZE kernel staging 跨 chunks 复用，禁止递归调用 userspace read/write 或按 count 分配 buffer。两个不同 OFD 的 offset 必须按 OFD identity 全局排序后整次持有，输出 `RegularFileWrite` 必须覆盖完整 copy；缺失前者会让反向 sendfile ABBA，缺失后者会让其他 writer 穿插。显式 offset 只推进 userspace value，空 offset 只推进输入 OFD；输出 OFD 始终按实际 written bytes 推进。同一 backing 的重叠区间和 O_APPEND 必须拒绝；socket/pipe zero-copy seam 未建立前返回 `EINVAL`，禁止伪造支持。
- regular-file cache miss fill、`fallocate`、write/append/truncate 与 mmap writeback 必须共用 page-cache operation lock；hit 只读 page index，miss 在取得 operation 后二次检查并唯一执行 storage read+publication，禁止旧 fill 穿过 mutation 后发布 stale page。`SharedFrame::bytes_mut` 只允许 page cache 在 `Arc<CachedPage>` publication 前直接填充独占 miss frame；其他 caller 不得用它绕过 shared-page 并发语义。ext2 完整对齐 storage block 必须直接读取到 caller buffer，只有 partial block 可使用 scratch。`page_cache::RegularFile` concrete facade 只允许 syscall regular I/O 与 task ELF loader 持有：syscall 每次调用解析一次并跨 user-copy chunks 复用，executable source 在完整解析生命周期复用；禁止恢复按 chunk/probe 传 inode 的 free-function 双轨入口。所有 regular write 必须先取得 `RegularFileWrite` 的 write_sequence gate，完整 scalar/vector syscall 只取得一次，并固定 write_sequence → operation 锁序；truncate、allocate、fd/global sync 使用同一外层顺序。page fault 与 AddressSpace 已持 mm lock 的 mapping sync 只能取得 operation，禁止获取 write_sequence，否则同文件 mmap user-copy 会自死锁或形成 mm → sequence / sequence → mm 反向锁序。`syscall::fs::io::regular::{read_vectors,write_vectors}` 是 scalar、vector 与 positioned handler 共用的唯一 chunk/user-copy/partial/RLIMIT engine；其 staging 固定为 PAGE_SIZE，与 page-cache 和 user-copy 页粒度一致，禁止恢复会令每页重复多次 facade/page-walk 的小块 policy。ordinary vector operation 必须整次持有一个 OFD offset lock，禁止递归调用 scalar syscall 在 iovec 间释放 ownership。ext2 只在同一 mutation/JBD2 owner 内分配 hole、更新 block pointers/i_blocks/i_size。write/append 提交 storage 后只能获取一次 page-index lock，并按 BTree range 只访问写区间内实际 resident page；禁止按输入页扫描 cache hole、重复取锁或 clone Arc。writeback 必须在 operation domain 内只取得一次稳定 EOF，以固定 32-entry scan batch 只 clone dirty Arc，并复用唯一 PAGE_SIZE stack scratch；禁止按 range 构造 Vec、按页读取 metadata 或分配临时 buffer，否则大 mapping munmap 会在内存压力下因 writeback 自身 OOM。CachedPage 的 dirty bit 与 writable-PTE count 必须由同一 atomic state 拥有，writable publication 必须原子设置 dirty+writer，零 writer 与 clear-dirty 只能在一次 CAS 中提交；拆成两个 Atomic 会吞掉并发 writer publication。`SharedPage` 不得恢复无 writer lifetime 的 `mark_dirty` 入口，否则 writeback 无法用同一 state 证明 clean transition。大 range 允许拆成有界且各自完整提交的 transaction，禁止 syscall 写零覆盖已有内容、伪造成功或维护第二份 allocation map。
- BSD `flock` state 只由 VFS 以 mounted inode identity 持有，OFD pointer 只作为 open-file-description lifetime identity；Task wait registry 只拥有 interruptible membership，VFS notifier 只投递发生变化的 key。唯一 VFS lock-table mutex 覆盖 shared/exclusive 转换，释放后才调用 notifier；缺失该锁会让两个 exclusive holder 同时成功，反向持锁或在锁内通知会与 wait publication 死锁。最后 descriptor close 必须经 OFD descriptor_refs 释放，禁止按 Process/fd 复制 lock state或让 ext2 维护第二套 advisory lock。
- POSIX record lock 只由 VFS 以 mounted inode identity + Process TGID 持有；同 owner 的相交 range 在一次锁内拆分、替换、排序并合并。`F_SETLKW` 只把 membership 交给统一 indexed wait owner，VFS 锁外通知；任一该 inode descriptor close、CLOEXEC close 或 Process exit 必须释放 Linux process-associated locks，禁止按 fd/OFD/ext2 复制 lock table。
- Process exit 只负责关闭本 Process 的 descriptor/OFD lifecycle；禁止在 exit/close cleanup 隐式执行全局 filesystem sync。durability 只能经 ext2 journal/writeback owner 与显式 `fsync/sync` seam 提交，否则任意短命进程都能阻塞全系统 I/O，并把 close 语义错误扩大为全局持久化屏障。
- procfs 与 `sysinfo` 必须消费 task façade 的同一采集边界；`/proc/stat` 的 `btime` 只投影 timer 唯一拥有的 boot epoch。syscall 只编码 Linux UAPI，禁止解析 `/proc` 文本、复制统计状态或在 ABI 层维护第二套 uptime/load/memory/task counter。
- `uname` 只投影 system module 的 immutable identity；`getcpu/riscv_hwprobe` 只通过 system façade 投影 HartTopology 的 logical CPU/platform facts；`sched_getaffinity` 只读 SchedulingState owner；`gettimeofday` 与 `clock_gettime(CLOCK_REALTIME)` 只投影 timer realtime owner。禁止 syscall module 维护 hostname、ISA、CPU mask、release、timezone 或第二份 wallclock offset。
- MMIO/volatile 只存在于 arch/driver HAL；user pointer 只通过 AddressSpace copy；磁盘 packed layout 只存在于 filesystem adapter。
- syscall memory handler 只解析 Linux flags/prot/errno；TaskControlBlock/AddressSpace 只持锁转发；VMA 选址、冲突、split/merge、frame rollback 与 PTE 提交只存在于 MemorySet。
- Process resource-limit owner 只在 task layer 执行权限、fork/exec lifecycle 与 CPU/NPROC policy；syscall 只编解码 `rlimit64`，memory/fs 只消费数值上限，不得反向读取 Process。
- frame allocator 的唯一慢路径通过 weak reclaimer seam 同步回收 `MADV_FREE`、clean private file/ELF 与无外部引用的 clean page-cache 页；private/shared dirty 页无 writeback 证明时禁止回收。page-cache reclaimer 不分配临时容器，达到 caller quota 必须立即停止有序遍历，禁止用 `retain` 在足额后继续扫描整个 cache。kernel heap 只从 frame allocator 转移页，不维护第二份物理容量 owner。
- `memory::frame_allocator` 的 external interface 只包含初始化、单页/连续分配、统计与 `FrameTracker` lifetime；具体 allocator、recycler mutation 和错误类型保持 private。有序 intrusive recycler 以页首 usize 存放严格递增 next PPN：single alloc O(1)，fresh contiguous O(1)，recycled contiguous 单次 O(recycled pages) 扫描。`FrameTracker` Drop 必须在一次 IRQ-safe lock 内先验证整个区间、再 batch splice，禁止逐页抢锁或失败后留下半回收 range；不得另建 bitmap/Vec 复制 free-state owner。
- futex private key 由 AddressSpace identity + uaddr 构成；anonymous/file shared key 只能由 MemorySet 从 backing identity + offset 归一化。IndexedWaitQueue 锁在 AddressSpace 锁外覆盖 key/value 比较、membership 发布、bitset wake 与 requeue，禁止 syscall、Process 或 scheduler 复制映射判断。
- task loader 是 pathname、Linux script rewrite 与 inode 到 `ExecutableSource` adapter 的唯一 owner；memory 只消费最终 ELF 随机读 seam，并唯一拥有 ELF 解析计划、PT_LOAD 映射、initial stack 与失败回滚。禁止恢复完整文件 `Vec`、filesystem 到 memory 的具体类型泄漏或第二套 script/ELF loader。
- thread-directed signal 发布到 Thread pending；kill/TTY/SIGCHLD 发布到 ProcessSignalState shared pending。两者经同一 delivery/wait seam 先发布 pending bit，再从 wait 的唯一 owner 注销 membership；blocking path 必须在 owner lock 内复查两类 pending，禁止 signal-before-enqueue lost wakeup。
- stop/continue 冲突消除必须与 signal queue generation 同锁；Process graph 唯一提交 group-stop/continue 与 parent event，scheduler 只维护每个 Thread 的 `StopPending/Stopped` membership。禁止用第二套 stopped flag 与 run state 人工同步。
- Process graph 在 exec point-of-no-return 唯一发布 `has_execed`，parent-side `setpgid` 必须在同一 graph lock 下得到成功或 `EACCES`。Process 退出/重父子关系前后只在该 owner 内计算 orphan transition，并冻结当时的 target TGID；锁外仅经统一 signal seam 发布全组 SIGHUP 后 SIGCONT。禁止在 syscall、Terminal 或 scheduler 复制 parent/SID/PGID/orphan 状态。
- global init 的 unkillable policy 位于统一 signal generation/delivery：默认 disposition signal 被丢弃，显式 handler 和 blocked pending 仍遵循 Linux 语义，强制同步 fault 不经该豁免。默认 SIGTSTP/SIGTTIN/SIGTTOU 在 delivery 时必须复查 orphan group，SIGSTOP 不豁免。
- Terminal 唯一判断 controlling session、foreground group 与 `TOSTOP`；TaskManager 唯一判断 caller process group、orphan 状态及 signal mask/disposition。后台访问只经该 seam 产生 SIGTTIN/SIGTTOU、EIO 或 syscall restart，禁止 syscall pathname/device 特判复制 job-control policy。
- session leader 退出使用固定 graph -> Terminal 锁序原子取走 controlling session/foreground PGID，并在 graph lock 内冻结 foreground target；SIGHUP 必须在锁外经统一 signal seam 发布，禁止按可能已复用的 PGID 重新选择目标。
- UART hardirq 不调度、不分配，只清空设备 FIFO并发布 console softirq；console read 在统一 indexed wait owner 内复查 RX ring，deferred consumer 才移除 membership 并 wake task。
- syscall handler 只能向 dispatcher 返回内部 restart 结果；trap layer 将其暂存为 `EINTR` 并把原 `a0..a5/a7/ecall PC` 交给当前 Thread。实际交付的 handler disposition 含 `SA_RESTART` 时才把 replay context 写入 signal frame，否则 frame 保留 `EINTR`；内部结果不得进入 U-mode。
- dispatcher 对未接入 number 只返回 `ENOSYS`，不得为每次未知调用打印日志；syscall number 与调用频率完全由 U-mode 控制，逐调用日志会把正常 capability/io_uring feature probe 变成 console I/O 放大与拒绝服务面。诊断必须使用显式临时探针并在完成后移除。
- Thread exit 发布顺序固定为 robust cleanup -> process graph removal -> clear-child-tid/futex wake；join completion 不得早于 Thread owner 注销。
- vfork caller 使用独立 `WaitMembership::Vfork(child)` 且不可被 signal 提前解除；child Process 与 parent 精确共享同一 AddressSpace Arc，并使用按 TID 分配的 supervisor trap page。只有 exec 原子替换 child handle并删除临时页，或 exit 先删除临时页，才能消费 child node 的唯一 caller waiter；不得挂起整个 parent Process。
- child exit/stop/continue event 必须在 process graph 锁内 claim；status copyout 成功后才消费，失败必须 release。多个 parent Thread 可分别等待，但同一 event 不得被重复返回或回收。
- private expedited membarrier registration 只存在于 AddressSpace；syscall 只能通过 task façade 请求 arch 对所有 active DTB hart 做同步 full fence。不得用本地 fence、RFENCE、吞 `ENOSYS` 或 musl fallback 冒充跨 hart memory ordering。
- terminal exit 必须先在可返回的 prepare frame 完成副作用并释放全部 task Arc，再以只含 raw context 的凭据切到 idle，由 per-hart deferred-reap slot 保活并析构 TCB；禁止从持有 task Arc 的 Rust frame 永久切走。
- user trap return 的 noreturn trampoline 前必须显式释放当前 TCB Arc；该 Rust frame 不会展开，依赖作用域析构会让每次 syscall 永久增加一个 kernel-stack 自引用。
- 首个 `exit_group` 或默认致命 signal 在 process graph 唯一提交 group-exit status；所有 sibling 只经 signal/wait/scheduler seam 回到自身内核栈退出，最后一个 Thread 才发布 zombie 与 SIGCHLD。禁止远程释放运行栈、重复保存 status 或把 signal death 改写成 shell exit code。
- raw CSR、DMA、page-table pointer、trap context 和 packed disk unsafe 必须有局部 `SAFETY:` 证明。
- 禁止 `static mut`、私有 syscall、固定 hart 容量、console syscall 旁路、deprecated/feature-flag 双轨。
- 禁止 `common/utils/helpers/misc/manager/base/shared/core` 等无领域含义的目录。
- `user/` 顶层只允许固定 BusyBox config、passwd/group identity、inittab/network service/udhcpc lease script、musl consumer 与 dynamic-loader C probe；围栏禁止 Rust user crate/source/linker、`build-user` 和旧 init artifact。

## 6. Change contract

修改前必须确定所属 module、状态 owner、interface 变化、依赖变化以及 error/exit/interrupt cleanup。依赖采用正向 allowlist，未列出的跨 module 依赖一律失败。扩大 interface 或新增 global state 必须更新对应权威契约；不得修改围栏来掩盖实现问题。只有其他架构规则全部通过时，`cargo run -p architecture-check -- --write-interface` 才能更新 interface contract。

唯一强制验证入口是 `make verify`。静态围栏、build 与 artifact gate 成功后，topology、musl、BusyBox 与 APK runtime gate 才可并发；每条 gate 必须从 `target/rootfs.img` 派生独占可写镜像并使用独立 success stamp，禁止读取或改写开发 `fs.img`；网络 fixture 必须使用不相交的内部 host port domain。recursive Make 等待全部 gate，任一失败都禁止进入最终 diff hygiene；禁止以并发为由跳过、抽样或提前发布成功证据。
